## 이슈 요약

첨부 Lottie asset과 결과 이미지만 제시된 rendering discrepancy다. 어떤 layer/property가 잘못됐는지 본문에 좁혀져 있지 않다.

## main 코드 조사

입구는 `src/loaders/lottie/tvgLottieParser.cpp`, model은 `src/loaders/lottie/tvgLottieModel.h`, scene 생성은 `src/loaders/lottie/tvgLottieBuilder.h`와 `src/loaders/lottie/tvgLottieBuilder.cpp`, 시간 평가는 `src/loaders/lottie/tvgLottieProperty.h`다. 그러나 asset을 실행·축소하지 않고는 어느 단계가 관련됐는지 특정할 근거가 없다.

```text
JSON bytes -> Parser -> Model/Property -> Builder/Paint tree -> SW | GL | WG
              ^          ^                ^                    ^
          값이 다른가?  frame 값?       scene graph?          backend만?
```

## 원인 가설

**불확실:** transform, mask, trim, text, effect 등 가능한 원인이 너무 많다. 이미지만 보고 특정 subsystem을 지목하는 것은 추측이다.

## 수정 방향 계획

첨부 asset을 current main과 lottie-web/reference player에서 같은 frame으로 비교하고, layer를 제거해 최소 재현 JSON을 만든다. 그 후 property parsing 결과와 builder가 만든 Paint tree를 비교한다.

## 위험/검증

최소 재현 없이 전체 asset만 기준으로 수정하면 다른 mask·trim·text 경로를 회귀시킬 수 있다. reference runtime/version과 비교 frame을 고정하고 CPU/GL/WG 결과를 각각 검증해야 한다.

## 추가 확인 필요

본문에 frame 번호, reference runtime/version, 문제 property가 없어 초기 triage 자체가 큰 작업이다. 최소 재현이 만들어진 뒤에는 난이도를 다시 평가할 수 있다.
