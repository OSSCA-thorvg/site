# #2211 — WebAssembly SIMD 지원

- **Link:** https://github.com/thorvg/thorvg/issues/2211
- **난이도:** 78/100
- **초심자 추천:** 비추천(benchmark 또는 한 함수 prototype은 조건부)
- **관련 영역:** SW raster SIMD, Emscripten, Meson feature flag, binary/runtime compatibility
- **배울 수 있는 것:** 128-bit lanes, vector alpha blend, tail 처리, 성능 측정
- **조사 기준:** `main@f989b27892bab31f224f810a54782055eba1e3bc`

## 이슈 요약

AVX/NEON 외에 WebAssembly SIMD 경로를 추가했을 때 SW renderer의 실제 성능 이득을 조사하고 구현하자는 최적화 이슈다. “지원”은 build flag만이 아니라 hot loop 선정, scalar와 정확도 일치, SIMD 비지원 runtime 정책과 wasm 크기까지 포함한다.

## main 코드 조사

### 확인된 증거

- `meson.build`의 `simd=true` 분기는 x86→AVX, ARM/AArch64→NEON만 정의한다. wasm cpu family 분기와 `-msimd128` option은 없다.
- 저장소에 `wasm_simd128.h` include나 `THORVG_*WASM*SIMD` macro가 없다.
- `tvgSwRaster.cpp`는 compile-time macro로 AVX/NEON/scalar를 선택한다.
- AVX/NEON 구현은 별도 header에 유사한 함수 세트를 두며, 일부 public raster helper에는 여전히 `TODO: Support SIMD accelerations`가 있다.
- SIMD fill은 vector loop 뒤에 scalar leftovers를 처리한다. WASM도 같은 tail 계약이 필요하다.

```cpp
// meson.build: wasm은 현재 어느 분기에도 들어가지 않는다.
if host_machine.cpu_family().startswith('x86')
  config_h.set10('THORVG_AVX_VECTOR_SUPPORT', true)
elif host_machine.cpu_family().startswith('arm')
  config_h.set10('THORVG_NEON_VECTOR_SUPPORT', true)
endif
```

### 아직 확인되지 않은 부분

- 현재 Emscripten compiler가 scalar loop를 어느 정도 auto-vectorize하는지 측정하지 않았다.
- 어떤 raster 함수가 WASM profile의 상위 병목인지 확인하지 않았다.
- SIMD binary를 별도 배포할지 baseline module 자체에 요구할지 정책이 없다.

## 원인 가설

- **확인됨:** ThorVG에는 wasm SIMD compile path와 intrinsic 구현이 없다.
- **가설:** `rasterPixel32`/`rasterGrayscale8` fill은 단순하고 검증 가능해 첫 prototype에 적합하지만, 전체 frame 성능 이득은 작을 수 있다.
- **가설:** alpha blend hot loop가 더 큰 이득을 낼 가능성이 있지만 AVX/NEON의 byte/word packing을 `v128` 연산으로 정확히 재구성해야 해 난도가 높다.

| 경로 | register | 현재 선택 | 핵심 검증 |
|---|---|---|---|
| Scalar | 없음 | fallback | 기준 결과 |
| AVX | 128/256-bit | x86 | unaligned/tail |
| NEON | 64/128-bit | ARM | AArch64 분기 |
| WASM SIMD | 128-bit | **없음** | `-msimd128`, runtime support, wasm size |

## 수정 방향과 실현 가능성

1. scalar WASM에서 대표 SVG/Lottie scene과 raster 함수별 profile/benchmark를 만든다.
2. `rasterPixel32` 또는 `rasterGrayscale8` 하나를 `wasm_simd128.h`로 구현해 compile flag와 dispatcher를 검증한다.
3. 다음 단계로 가장 뜨거운 alpha blend 하나를 옮기고 모든 alpha/length/alignment 입력을 scalar와 differential test한다.
4. SIMD on/off의 render time, FPS, `.wasm` 크기를 같은 Emscripten 최적화 설정에서 기록한다.
5. SIMD 미지원 runtime을 위해 두 binary를 제공할지 feature detection 후 선택할지 문서화한다.

```text
len pixels = [ v128 ][ v128 ] ... [tail]
             SIMD hot loop          scalar fallback
```

## 위험과 검증

- intrinsic을 추가했어도 compiler가 scalar보다 느린 코드를 만들 수 있으므로 측정이 완료 조건이다.
- 255 나눗셈 근사와 premultiplied alpha는 scalar와 bit-exact여야 한다.
- Emscripten threaded/non-threaded, debug/release, SIMD 지원/미지원 runtime을 구분한다.
