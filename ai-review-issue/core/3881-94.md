# #3881 API: runtime memory allocator customization

- Link: https://github.com/thorvg/thorvg/issues/3881
- 난이도: 94/100
- 실현 가능성: 낮음
- 초심자 추천: 비추천
- 관련 영역: allocator callbacks, Initializer, global new/delete, CAPI/ABI
- 배울 수 있는 것: allocator pairing, initialization generation, realloc/alignment contract

## 이슈 요약

사용자 allocator callbacks를 runtime에 등록해 embedded pool이나 memory tracking을 지원하자는 API다. wrapper는 있지만 모든 allocation/free가 같은 allocator generation을 사용하도록 보장해야 한다.

## main 코드 조사

**확인된 증거**

- `tvgAllocator.h`의 template wrappers는 현재 inline으로 `std::malloc/calloc/realloc/free`를 호출한다.
- `tvgInitializer.cpp`는 global `operator new/new[]/delete/delete[]`도 이 wrapper로 연결한다.
- public `Initializer::init(threads)`와 CAPI `tvg_engine_init(threads)`에는 allocator 인자가 없다.
- bundled third-party와 GPU/OS objects까지 wrapper가 포괄한다는 보장은 없다.

```cpp
template<typename T = void>
static inline T* malloc(size_t size) {
    return static_cast<T*>(std::malloc(size));
}

void* operator new(std::size_t size) { return tvg::malloc(size); }
```

```text
init allocator A -> allocate p(A) -> nested init allocator B? -> free p(?)
                              allocator generation이 바뀌면 pairing 파괴
```

## 원인 가설과 확인 방법

- **확정:** wrapper seam은 있으나 runtime function table/lifetime contract는 없다.
- **가설:** indirect callback 비용은 acceptable하고 custom pool이 유의미한 이득을 준다. 측정되지 않았다.
- **확인 방법:** default callback prototype으로 allocation-heavy SVG/Lottie workload의 calls/latency/binary size를 baseline과 비교한다.

## 수정 방향 계획

1. callbacks의 alignment, zero-size, failure, realloc와 user-data contract를 명세한다.
2. 최초 init 전에만 등록하거나 allocator generation을 object lifetime까지 보존하는 모델을 선택한다.
3. nested init은 같은 callbacks만 허용하고 final term 후 outstanding allocation 정책을 정한다.
4. CAPI는 version/size가 있는 allocator descriptor로 확장 가능성을 설계한다.

## 실현 가능성 판단

기술적으로 가능하지만 heap corruption 위험과 ABI 결정 때문에 **낮음**이다. 초심자에게 적합하지 않다.

## 위험/검증

- counting/failing/aligned/reentrant allocator와 multi-thread stress를 수행한다.
- allocation과 free callback pairing을 ID로 검증한다.
- global new symbol 영향, exception-disabled OOM, static/shared/WASM을 검사한다.
