## 이슈 요약

작은 closed path에 큰 Round stroke와 10배 transform을 적용하면 SW에 hole이 생기고 GL/WG와 결과가 달라지는 문제다. 재현 code가 짧고 SW-only 분류가 있어 조사 가능성이 높지만 wide offset geometry 수정은 위험하다.

## main 코드 조사

### 확인된 증거

- SW는 left/right offset border를 만들고 `strokeExportOutline()`이 두 contour를 `FillRule::NonZero`로 내보낸다.
- `_inside()`는 두 선분 길이가 충분할 때만 intersection을 쓰고 아니면 단순 offset point를 추가한다.
- `_beginSubPath()`는 **Round join + closed path**에서 `handleWideStrokes=false`가 된다.
- GL/WG는 별도 triangle tessellator를 사용하므로 SW offset-outline과 같은 topology algorithm이 아니다.
- 이슈 표현은 “even/odd hole”이지만 current SW exported rule은 EvenOdd가 아닌 NonZero다.

```cpp
if ((stroke.join != StrokeJoin::Round) ||
    (!stroke.closedSubPath && stroke.cap == StrokeCap::Butt))
    stroke.handleWideStrokes = true;
else
    stroke.handleWideStrokes = false;

outline->fillRule = FillRule::NonZero;
```

### 아직 확인되지 않은 부분

- current main의 exact hole pixels와 exported contour orientation을 실행 dump하지 않았다.
- `handleWideStrokes` 조건이 직접 원인인지, `_inside()`의 non-intersect branch인지 미확정이다.
- transform을 path에 bake한 경우와 renderer matrix로 적용한 경우가 같은지 비교하지 않았다.

## 원인 가설

- **강한 가설:** inner border가 path를 넘어 self-intersect/방향 역전하고 NonZero winding에서 일부 영역을 상쇄한다.
- **코드 단서:** Round closed path가 wide-stroke 특수 처리에서 제외되는 조건은 이 예제와 정확히 겹친다.
- **대안 가설:** acute corner의 `_inside()` fallback point 순서만 틀려 contour orientation이 깨진다.

| backend | stroke 표현 | hole 후보 |
|---|---|---|
| SW | 두 offset contour → NonZero RLE | self-intersection/winding 상쇄 |
| GL | stroke triangle list | 별도 tessellation, reference |
| WG | stroke triangle list | 별도 tessellation, reference |

## 수정 방향과 실현 가능성

1. example을 CPU regression으로 만들고 width/segment ratio를 sweep해 최초 hole을 고정한다.
2. left/right border를 SVG/debug path로 export해 point order, signed area와 self-intersection을 표시한다.
3. transform bake vs matrix 적용, CW/CCW path를 비교한다.
4. Round closed-path wide handling 또는 inner border branch 중 최초 잘못된 단계만 최소 수정한다.
5. open/closed, 모든 join/cap, acute/obtuse, width 0~수백 배와 GL/WG visual을 검사한다.

## 위험과 검증

- fill rule을 EvenOdd로 바꾸는 식의 우회는 정상 overlap 의미를 바꿀 수 있다.
- self-intersection point 제거는 contour continuity와 join coverage를 깨뜨릴 수 있다.
- #2084의 huge-width 성능/overflow 문제와 visual topology 문제를 같은 fix로 섞지 않는다.
