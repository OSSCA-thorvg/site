# #2937 sw_engine: data race with openmp

- Link: https://github.com/thorvg/thorvg/issues/2937
- 난이도: 64/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 관련 영역: CPU post effect, OpenMP, ThreadSanitizer, compositor buffer
- 배울 수 있는 것: 병렬 메모리 분할, barrier, sanitizer 오탐 판별, Gaussian blur

## 이슈 요약

Gaussian blur를 TSan+OpenMP로 실행할 때 worker write와 다음 read 사이 race가 보고된 이슈다. 현재 코드 모양만 보면 행은 분리되고 `parallel for` 종료 barrier가 있어, 실제 race인지 OpenMP runtime을 TSan이 인식하지 못한 것인지 재현으로 구분해야 한다.

## main 코드 조사

**확인된 증거**

- `_gaussianFilter()`는 `#pragma omp parallel for`로 `y`를 나누며 각 iteration은 `dst[(y * stride) * 4 ...]`를 쓴다.
- `nowait`가 없으므로 정상 OpenMP 의미에서는 loop 종료에 implicit barrier가 있다.
- `effectGaussianBlur()`는 함수 호출이 끝난 뒤에만 front/back을 swap하고 다음 pass를 시작한다.
- vertical pass는 `rasterXYFlip()` 후 width/height를 바꿔 같은 filter를 호출하며, `SwRenderer::request(..., true)`는 정사각 임시 surface를 요청한다.
- Meson의 OpenMP dependency는 optional이어서 지원되지 않는 compiler에서는 pragma가 무시될 수 있다.

```cpp
#pragma omp parallel for
for (int y = 0; y < h; ++y) {
    auto i = y * stride * 4;
    // 이 iteration은 자신의 dst row를 기록한다.
}
// OpenMP 의미상 여기에는 implicit barrier가 있다.
```

```text
horizontal filter ─barrier─▶ swap ─▶ next filter
flip ─▶ vertical filter ─barrier─▶ swap ─▶ flip back
```

## 원인 가설과 확인 방법

- **확정:** 소스에 명시적 `nowait`는 없고 row 시작 주소는 `y * stride`로 분리된다.
- **가설 1:** libgomp/libomp의 barrier를 해당 TSan 조합이 인식하지 못한 오탐이다.
- **가설 2:** flipped bbox/stride 전제가 깨지는 특정 크기에서 row 범위가 실제로 겹친다.
- **확인 방법:** 경고 stack, runtime 버전, 각 worker의 `[dst begin,end)`를 기록하고 범위 교집합을 검사한 뒤 동일 구조의 최소 OpenMP 프로그램과 비교한다.

## 수정 방향 계획

1. 현재 main에서 clang/libomp와 gcc/libgomp 조합을 기록하며 동일 fixture를 반복한다.
2. OpenMP on/off와 한 thread/여러 thread 결과 및 경고를 비교한다.
3. 실제 overlap이면 bbox/stride 계산을 고치고 주소 불변식을 test한다.
4. barrier 인식 문제면 mutex를 넣지 말고 지원 toolchain 문서, runtime upgrade 또는 좁은 suppression을 제안한다.

## 실현 가능성 판단

**중간**이다. 재현 환경만 확보하면 조사 절차는 명확하며 초심자도 최소 reproducer를 만들 수 있다. 다만 race가 아니라는 결론도 happens-before와 주소 범위 증거가 있어야 하므로 concurrency 멘토가 필요하다.

## 위험/검증

- 경고 제거만을 위한 mutex/serial fallback은 성능 회귀 때문에 피한다.
- TSan 외에 pixel 동일성, ASan/UBSan, OpenMP on/off와 blur benchmark를 확인한다.
- WASM/embedded/non-OpenMP 빌드에는 새 OpenMP symbol 의존성이 생기지 않아야 한다.
