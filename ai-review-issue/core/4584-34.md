## 이슈 요약

첨부 SVG의 label은 다음처럼 element 이름을 text로 표시한다.

```xml
<text ...>pattern with &lt;image&gt;</text>
```

expected renderer는 `pattern with <image>`를 표시하지만 ThorVG current 결과는 `pattern with &lt;image&gt;`를 문자 그대로 표시한다. 그 결과 text advance와 screenshot crop도 달라진다. 첨부 파일을 직접 확인하면 pattern rendering보다 XML entity decoding 문제임을 분리할 수 있다.

## main 코드 조사

[`xmlParse()`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/loaders/svg/tvgXmlParser.cpp#L409-L432)은 `<` 전까지의 원본 byte 범위를 `XMLType::Data` callback에 그대로 전달한다. text 안에서는 whitespace stripping도 의도적으로 건너뛴다.

SVG callback [`_svgLoaderParserText()`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/loaders/svg/tvgSvgLoader.cpp#L3394-L3408)은 받은 content를 parent text 또는 새 tspan run에 그대로 append한다. 이 두 단계 사이에 entity reference를 Unicode text로 바꾸는 처리가 없다.

parser에는 `&amp;`, `&lt;`, `&gt;` 등의 목록이 있지만 현재 helper는 tag/attribute 경계의 공백을 건너뛰는 용도로 사용된다. XML text content의 replacement text를 생성하는 decoder는 아니다.

## 원인 가설

- **확인:** `XMLType::Data`가 `&lt;`와 `&gt;`를 포함한 raw slice를 전달한다.
- **확인:** SVG text builder도 raw slice를 그대로 `Text::text()`까지 보낸다.
- **확인:** XML 1.0은 predefined entity reference를 해당 문자 replacement text로 처리한다.
- **추론:** parser가 low-allocation slice API를 유지하면서 attribute 경계 처리만 구현했고, text node의 entity expansion 단계가 빠진 것이다.

## 수정 방향 계획

1. XML parser에 data용 decoder를 추가해 `amp`, `lt`, `gt`, `apos`, `quot`와 decimal/hex character reference를 UTF-8로 변환한다.
2. decode는 parser가 실제 markup `<`를 기준으로 token을 나눈 **뒤** 수행한다. `&lt;`가 `<`로 바뀌어도 새 tag로 재해석하면 안 된다.
3. `XMLType::CData`는 entity를 확장하지 않고 원문을 유지한다.
4. tspan 사이에서 data callback이 여러 번 와도 문자 순서와 공백이 보존되도록 decoded chunk를 append한다.
5. 알 수 없는 entity, 빠진 semicolon, invalid code point의 loader 실패/보존 정책을 XML parser의 기존 오류 정책에 맞춰 명시한다.
6. 첨부 SVG의 작은 축약 fixture와 parser-level 단위 테스트를 추가한다.

## 초심자 시작 가이드

render screenshot 전에 다음 문자열을 parser test로 고정한다.

| 입력 | text 결과 |
|---|---|
| `A &lt; B &amp;&amp; B &gt; C` | `A < B && B > C` |
| `&quot;x&quot; &apos;y&apos;` | `"x" 'y'` |
| `&#60; &#x3E;` | `< >` |
| `<![CDATA[&lt;]]>` | `&lt;` |
| `A<tspan>&lt;B&gt;</tspan>C` | run을 합쳐 `A<B>C` |

decoder를 attribute parser의 기존 skip helper에 억지로 합치지 말고, “경계를 찾는 기능”과 “replacement text를 만드는 기능”을 분리하는 것이 안전하다.

## 위험/검증

- predefined entity와 decimal/hex numeric reference를 모두 검사한다.
- non-ASCII code point를 올바른 UTF-8로 만들고 surrogate·범위 밖 code point를 거부한다.
- CDATA와 escaped text를 중복 decode하지 않는다.
- `xml:space="default"`/`preserve`, leading/trailing whitespace와 여러 tspan 조합을 확인한다.
- entity가 들어간 attribute의 기존 동작과 일반 SVG loading이 회귀하지 않는지 검사한다.
- 첨부 sample을 expected renderer와 pixel 비교해 label 폭과 위치가 복구되는지 확인한다.
