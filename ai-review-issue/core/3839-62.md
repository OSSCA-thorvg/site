- Link: https://github.com/thorvg/thorvg/issues/3839
- 난이도: 62/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 관련 영역: keyframe parser, interpolator cache, vector frame
- 배울 수 있는 것: cubic easing, component interpolation, template model, regression fixture

## 이슈 요약

vector property의 easing handle이 차원별 배열일 때 ThorVG가 첫 값만 보존해 모든 component에 같은 progress를 적용하는 문제다. 원인 위치는 명확하지만 parser와 frame storage/interpolation을 함께 바꿔야 한다.

## main 코드 조사

**확인된 증거**

- `getInterpolatorPoint()`는 `x/y`를 각각 scalar `Point` 한 성분으로 읽는다. 배열 입력이면 `getValue(float&)`가 첫 값만 취한다.
- `parseKeyFrame()`은 keyframe당 `LottieInterpolator*` 하나만 만든다.
- `LottieVectorFrame::interpolate()`도 한 progress `t`를 Point 전체 `lerp` 또는 spatial Bezier에 쓴다.
- cache key는 in/out의 네 scalar만 포함해 차원별 curve를 표현하지 못한다.

```cpp
struct LottieVectorFrame {
    LottieInterpolator* interpolator;  // component별이 아님
    T interpolate(LottieVectorFrame* next, float frameNo) {
        auto t = interpolator->progress(...);
        return tvg::lerp(value, next->value, t);
    }
};
```

```text
입력: x easing [x0,x1], y easing [y0,y1]
현재: x0/y0 한 curve ─▶ Point 전체
필요: component별 curve ─▶ x/y 각각 progress
```

## 원인 가설과 확인 방법

- **확정:** parser와 frame model이 multidimensional easing 값을 폐기한다.
- **미확정:** spatial tangent가 있을 때 component별 time progress를 path parameter와 어떻게 결합할지 reference 확인이 필요하다.
- **확인 방법:** 동일 keyframe을 spatial tangent 유/무로 나눠 reference positions를 frame별 고정한다.

## 수정 방향 계획

1. scalar easing과 2D/3D easing fixture의 reference 좌표를 고정한다.
2. parser가 scalar/array를 구분해 차원별 handle을 보존하도록 한다.
3. 기존 scalar fast path는 pointer 하나를 유지하고 multidimensional frame만 추가 storage를 사용한다.
4. non-spatial component lerp를 먼저 구현한 뒤 spatial semantics는 별도 test를 통과할 때 확장한다.

## 실현 가능성 판단

**중간**이며 초심자 조건부 추천이다. parser-only 수정은 불완전하므로 property model을 이해하는 멘토 review가 필요하다.

## 위험/검증

- frame/interpolator ownership, cache 중복과 `LottieVectorFrame` 크기 증가를 확인한다.
- scalar easing의 결과와 성능이 변하지 않아야 한다.
- Point, Point3, separated dimensions와 spatial tangent를 회귀 검사한다.
