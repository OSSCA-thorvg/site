## 이슈 요약

현재 `-Dlog=true/false`가 `TVGERR`와 `TVGLOG`를 함께 켜고 끄므로 error만 남기는 build-time level을 제안한다. warn/info 등 새 level까지 넓히는 내용도 포함한다.

## main 코드 조사

`meson_options.txt`의 `log` boolean을 top-level `meson.build`가 읽어 `THORVG_LOG_ENABLED`를 만든다. `src/common/tvgCommon.h`는 이 define 하나 아래에서 `TVGERR`와 `TVGLOG`를 동시에 정의한다. 수정 축이 좁고 build matrix로 검증할 수 있다.

```cpp
#ifdef THORVG_LOG_ENABLED
    #define TVGERR(...)
    #define TVGLOG(...)
#else
    #define TVGERR(...) do {} while (0)
    #define TVGLOG(...) do {} while (0)
#endif
```

## 원인 가설

**확인된 구조:** 현재 config가 logging 전체를 한 boolean로 모델링해 severity별 compile-time 제거가 불가능하다.

## 수정 방향 계획

1. 범위를 먼저 “error-only 지원”으로 좁히고 기존 `-Dlog` 호환 정책을 합의한다.
2. Meson enum/string option을 config define으로 변환한다.
3. `TVGERR`/`TVGLOG`를 level별로 독립 활성화한다.
4. 각 option에서 작은 compile/run test로 stdout/stderr와 제거 여부를 확인한다.

## 위험/검증

Issue 예시의 `type: 'array'`와 scalar `value: 'all'` 조합은 Meson schema를 다시 확인해야 한다. warn/info까지 한 번에 추가하면 전 호출부와 naming 정책으로 범위가 커진다. 기존 option을 깨지 않는 작은 단계라면 좋은 초심자 build 과제다.
