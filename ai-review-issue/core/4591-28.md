## 이슈 요약

공개 헤더의 `@param` 설명에서 “The default value is X.”가 C++ 기본 인자와 새 객체의 초기 속성값을 모두 가리킨다. 예를 들어 `Shape::fill(r, g, b, a = 255)`에서 `a`만 생략할 수 있지만 네 채널 모두 같은 “default value” 문구를 사용한다. 호출자가 생략할 수 있는 값인지, 객체 생성 직후 이미 설정된 값인지 구별하기 어렵다.

C API에는 언어 차원의 기본 인자가 없는데도 같은 표현이 setter와 getter의 출력 매개변수에 남아 있어 모호성이 더 크다.

## main 코드 조사

현재 [`inc/thorvg.h`](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/inc/thorvg.h)에는 정확한 문구 `The default value is`가 17회, [`thorvg_capi.h`](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/src/bindings/capi/thorvg_capi.h)에는 21회 나온다.

C++의 [`Shape::strokeFill()` 선언](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/inc/thorvg.h#L1327-L1344)은 `r`, `g`, `b`, `a` 모두에 기본값 문구를 붙이지만 선언상 기본 인자는 `a = 255`뿐이다. 반면 [`Canvas::add()`](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/inc/thorvg.h#L914-L937)의 `at = nullptr`처럼 실제로 생략 가능한 매개변수에도 같은 문구가 쓰인다. 따라서 전역 문자열 치환만으로는 의미를 보존할 수 없다.

C API의 [`tvg_shape_set_stroke_color()`와 getter](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/src/bindings/capi/thorvg_capi.h#L1486-L1514)는 모든 인자를 명시해야 한다. 특히 getter의 `@param[out]`에도 기본값 문구가 있어, 이것이 호출 전 출력 버퍼의 값인지 shape의 초기 속성인지 분명하지 않다. [`fill color` setter/getter](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/src/bindings/capi/thorvg_capi.h#L1650-L1676)도 같은 패턴이다.

구현은 별도의 “초기 상태”를 가진다. [`RenderStroke`](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/src/renderer/tvgRender.h#L348-L361)는 width `0`, miter limit `4`, square cap, bevel join으로 초기화되고, [`Paint` transform 초기화](https://github.com/thorvg/thorvg/blob/6bf5bc97923bab4a778070074df69daf3430c2ed/src/renderer/tvgPaint.h#L240-L243)는 scale을 `1`로 둔다. 이 값들은 함수 호출에서 인자를 생략하는 규칙이 아니라 새 객체의 상태다.

C++ 표준의 [default arguments 절](https://eel.is/c++draft/dcl.fct.default)은 매개변수 선언의 initializer가 후행 인자를 생략한 호출에 사용되는 경우를 기본 인자로 정의한다. Doxygen의 [`@param` 명령](https://www.doxygen.nl/manual/commands.html#cmdparam)은 매개변수 방향과 설명을 문서화하지만 “default”의 의미를 대신 결정하지 않는다.

## 원인 가설

- **확인:** 같은 문구가 실제 C++ 기본 인자와 객체의 초기 렌더 상태에 모두 사용된다.
- **확인:** C API 함수 선언은 해당 값을 생략할 수 없고, 일부 `@param[out]`에도 기본값 문구가 있다.
- **확인:** scale, stroke width, miter limit, cap, join의 초기 상태는 renderer 내부 초기화에서 확인된다.
- **추론:** C++ 문서를 C API로 옮기거나 여러 setter/getter 설명을 맞추는 과정에서 “default argument”와 “initial property value”의 구분이 사라진 것으로 보인다.
- **추론:** 한 표현을 모두 삭제하는 것보다 의미별 용어 규칙을 정한 뒤 C++/C 대응 항목을 함께 수정해야 재발 가능성이 낮다.

## 수정 방향 계획

1. 두 공개 헤더에서 `default value`, `(default)`, `default is` 등 관련 표현을 전부 수집한다.
2. 각 항목을 다음 네 범주로 분류한다.
   - 실제 C++ 기본 인자: “If omitted, … is used.”
   - 새 객체의 초기 속성: “The initial … of a newly created object is …”
   - getter 출력값: 성공 시 반환되는 상태를 설명하고 기본값 표현은 제거
   - 권장 설정값: “recommended value”처럼 권장과 초기 상태를 분리
3. C++와 C API의 대응 함수는 같은 개념 용어를 사용하되, C API에는 “인자 생략” 표현을 사용하지 않는다.
4. 선언이나 구현은 바꾸지 않고 문서만 수정한다. 조사 중 실제 동작과 문서 값이 다르면 별도 동작 이슈로 분리한다.
5. 공개 헤더에 모호한 정확 문구가 다시 들어오지 않도록 가벼운 정적 검사나 리뷰 체크리스트를 추가한다.

## 초심자 시작 가이드

먼저 문구를 기계적으로 바꾸지 말고 목록을 만든다.

```sh
rg -n "The default value is|\\(default\\)|default is" \
  inc/thorvg.h src/bindings/capi/thorvg_capi.h
```

각 결과 옆에 `default-argument`, `initial-state`, `output`, `recommendation` 중 하나를 표시한다. 실제 기본 인자는 함수 선언의 `= value`로 확인하고, 초기 상태는 `tvgPaint.h`, `tvgRender.h` 같은 구현 초기화와 getter 결과로 교차 확인한다. 가장 명확한 한 쌍의 C++/C API부터 문구를 고친 뒤 같은 기준을 나머지 항목에 적용한다.

## 위험/검증

- 두 헤더에서 모호한 정확 문구와 변형 표현이 남지 않았는지 검색한다.
- C++ 선언의 실제 기본 인자(`= value`)가 문서에서 “생략 가능”으로 정확히 표현되는지 확인한다.
- C API 문서가 인자를 생략할 수 있다는 인상을 주지 않는지 확인한다.
- getter의 `@param[out]`에는 출력 계약과 실패 시 동작만 설명하고 호출 전 버퍼값을 암시하지 않는지 확인한다.
- C++/C 대응 함수의 초기 상태 값과 용어가 일치하는지 표로 대조한다.
- Doxygen을 생성해 링크, `@p`, `@c`, 줄바꿈과 경고가 정상인지 확인한다.
- 사이트 검증에서 Markdown 링크와 표가 정상적으로 처리되는지 확인한다.
