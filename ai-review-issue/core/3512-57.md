# #3512 — 90도 회전 이미지의 첫 pixel line 손실

- **Link:** https://github.com/thorvg/thorvg/issues/3512
- **난이도:** 57/100
- **초심자 추천:** 조건부(pixel fixture·trace부터 추천)
- **관련 영역:** SW image texmap, scanline bounds, inverse UV sampling
- **배울 수 있는 것:** pixel-center, half-open interval, float→int 경계
- **조사 기준:** `main@f989b27892bab31f224f810a54782055eba1e3bc`

## 이슈 요약

PNG를 translate한 뒤 정확히 90도 회전하면 첫 pixel line이 사라지는 SW renderer 문제다. 재현 코드가 짧고 직각 회전이라는 강한 경계 조건이 있어 조사 진입점은 좋지만, 임의 epsilon fix는 반대 edge 중복을 만들 수 있다.

## main 코드 조사

### 확인된 증거

- rotation image는 direct/scaled copy가 아니라 polygon texmap 경로로 간다.
- `utilBBox()`는 transformed bbox를 floor(min)/ceil(max)한 half-open region으로 만든다.
- `tvgSwRasterTexmap.h`는 edge의 float `_xa/_xb`를 `int32_t`로 잘라 `x1/x2`를 만들고 `dx = 1 - (_xa - x1)`로 UV를 pre-step한다.
- 계산된 `uu/vv`를 unsigned range와 비교하므로 작은 negative 값도 매우 큰 unsigned가 되어 pixel을 건너뛴다.

```cpp
x1 = std::max((int32_t)_xa, bbox.min.x);
dx = 1 - (_xa - x1);
u = _ua + dx * _dudx;
v = _va + dx * _dvdx;
if ((uint32_t)uu >= image.w || (uint32_t)vv >= image.h) continue;
```

### 아직 확인되지 않은 부분

- issue sample image는 local fixture로 저장돼 있지 않아 현재 main output을 직접 pixel dump하지 않았다.
- 첫 line에서 UV가 `-epsilon`, 정확히 `w/h`, 또는 scanline 자체가 비는지 미확정이다.

## 원인 가설

- **강한 가설:** 90도 inverse transform에서 첫 sample UV가 경계 밖으로 한 ulp 벗어나 unsigned bounds check에 걸린다.
- **대안 가설:** `_xa/_xb` truncation과 `dx=1-...`가 pixel center를 한 칸 안쪽부터 시작시킨다.
- bbox floor/ceil도 후보지만 texmap pre-step과 UV check가 더 직접적인 코드 근거다.

```text
destination pixel center -> inverse transform -> source UV
first row:                     -ε or h      -> bounds skip -> 빈 한 줄
```

## 수정 방향과 실현 가능성

1. 3×2 고유색 raw image로 0/90/180/270도와 정수/음수 translate test를 만든다.
2. 누락 row의 `_xa/_xb`, `x1/x2`, `u/v`, `uu/vv`를 기록해 최초 잘못된 값만 특정한다.
3. destination pixel-center 규약을 문서화하고 edge setup 또는 pre-step을 그 규약에 맞춰 수정한다.
4. nearest/bilinear, flip, non-integer rotation, clipping/mask와 stride>w를 회귀 검사한다.

| backend | image rotation 경로 | 이슈와의 관계 |
|---|---|---|
| SW | CPU polygon scanline + inverse UV | 직접 후보 |
| GL | GPU texture sampling | 비교 기준 |
| WG | GPU texture sampling | 비교 기준 |

## 위험과 검증

- UV epsilon clamp는 반대 edge pixel을 중복할 수 있으므로 원칙 없는 fix를 피한다.
- source/destination의 half-open 구간과 bilinear 마지막 texel 처리도 함께 고정한다.
- ASan으로 edge pixel read가 source 범위를 넘지 않는지 확인한다.
