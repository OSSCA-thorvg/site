## 이슈 요약

macOS Apple Silicon의 Apple Clang 환경에서 기본 `meson setup`은 성공하지만, 이후 compile 중 `omp.h`를 찾지 못해 실패한다. OpenMP option은 켜져 있는데 Meson dependency는 optional이라, config header가 지원을 선언한 상태와 compiler/linker 설정이 어긋난다.

## main 코드 조사

[`extra` option](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/meson_options.txt#L66-L70)은 array이고 기본값에 `openmp`를 포함한다. top-level [`meson.build`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/meson.build#L138-L149)는 option 문자열만 보고 `THORVG_OPENMP_SUPPORT`를 설정한다.

반면 CPU renderer의 [`meson.build`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/renderer/cpu_engine/meson.build#L21-L30)는 다음처럼 dependency 탐색 실패를 허용한다.

```meson
if openmp
    omp_dep = dependency('openmp', required: false)
endif
```

그 상태에서도 [`tvgSwRenderer.cpp`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/renderer/cpu_engine/tvgSwRenderer.cpp#L23-L29)는 config macro 때문에 `omp.h`를 include한다. Meson 문서상 `required: false`로 찾지 못한 dependency object는 target에 전달할 수 있지만 compile/link flag를 제공하지 않는다.

## 원인 가설

- **확인:** 기능 활성 여부와 dependency 발견 여부가 서로 다른 조건으로 계산된다.
- **확인:** 현재 `extra`는 `auto`가 아니라 사용자가 포함/제외하는 명시적 array option이다.
- **추론:** 따라서 `openmp`가 포함된 경우 dependency 미발견을 configure error로 만드는 것이 현재 option 의미와 가장 잘 맞는다.

## 수정 방향 계획

최소 수정은 다음과 같다.

```meson
if openmp
    omp_dep = dependency('openmp')
endif
```

1. OpenMP가 요청되면 dependency를 required로 탐색한다.
2. 찾지 못하면 `meson setup` 단계에서 설치 또는 `-Dextra=...`로 비활성화하라는 명확한 오류를 낸다.
3. macOS build 문서에 Apple Clang은 별도 OpenMP runtime이 필요할 수 있음을 적고, 정확한 설치 명령은 지원하는 package manager 범위가 정해질 때만 추가한다.
4. 장기적으로 자동 탐색이 필요하다면 `extra` array와 별개로 Meson `feature` option(`enabled`/`disabled`/`auto`)으로 옮기는 RFC를 분리한다.

## 초심자 시작 가이드

세 개의 깨끗한 build directory로 동작을 비교한다.

```sh
# dependency 있음: 성공
meson setup build-openmp

# OpenMP 명시 제외: dependency가 없어도 성공해야 함
meson setup build-no-openmp -Dextra=lottie_exp

# dependency 없음 + OpenMP 활성: setup에서 실패해야 함
meson setup build-missing-openmp
```

마지막 case가 compile까지 진행하지 않고 dependency 이름과 해결 방법을 보여 주는지 확인한다.

## 위험/검증

- macOS Apple Clang에서 libomp 설치 전·후 setup 결과를 확인한다.
- Linux GCC/Clang에서 OpenMP compile·link와 runtime test가 유지되는지 확인한다.
- `-Dextra=lottie_exp`, `-Dextra=[]` build가 `omp.h`를 include하지 않는지 확인한다.
- Android/iOS/WASM 등 OpenMP를 쓰지 않는 toolchain의 기존 CI option을 점검한다.
- 기본 option이 OpenMP를 포함한다는 사실이 의도인지 release/packaging 정책도 확인한다.
