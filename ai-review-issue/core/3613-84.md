# #3613 — SW straight-alpha mode 최적화

- **Link:** https://github.com/thorvg/thorvg/issues/3613
- **난이도:** 84/100
- **초심자 추천:** 비추천(conversion benchmark는 조건부)
- **관련 영역:** SW pixel pipeline, premultiplied/straight alpha, partial rendering, SIMD
- **배울 수 있는 것:** source-over 표현, rounding, full-surface conversion 비용
- **조사 기준:** `main@f989b27892bab31f224f810a54782055eba1e3bc`

## 이슈 요약

SW renderer는 내부 premultiplied 합성 후 `ABGR/ARGB8888S` target 전체를 straight alpha로 unpremultiply한다. 웹/native system이 추가 composition할 때 이 전환 비용을 줄이자는 최적화다. 내부 표현 전체를 straight로 바꾸는 것과 마지막 conversion만 최적화하는 것은 난이도가 크게 다르다.

## main 코드 조사

### 확인된 증거

- `SwRenderer::target()`은 S/non-S와 무관하게 surface를 `premultiplied=true`로 시작한다.
- `postRender()`는 S colorspace이면 `rasterUnpremultiply(surface)`를 호출한다.
- 해당 함수는 stride를 따라 **surface 전체 w×h**를 scalar loop하며 `OPTIMIZE_ME: +SIMD`가 있다.
- input image가 straight이면 `SwImageTask::run()`에서 premultiply한다.
- blend/mask의 많은 연산은 premultiplied channel을 전제로 하고 일부 custom blend에서 pixel별 unpremultiply도 수행한다.

```cpp
if (surface->cs == ColorSpace::ABGR8888S ||
    surface->cs == ColorSpace::ARGB8888S) {
    rasterUnpremultiply(surface);  // 현재 full surface
}
```

### 아직 확인되지 않은 부분

- 720p/1080p/4K와 scene 복잡도별 conversion 비중을 profile하지 않았다.
- partial rendering에서 이전 straight pixel과 새 premultiplied pixel이 어떤 시점에 다시 일관된 표현이 되는지 별도 correctness audit가 필요하다.
- web consumer가 실제로 straight buffer를 요구하는지, premultiplied target 선택으로 제거 가능한 workload 비율이 없다.

## 원인 가설

- **확인됨:** S target은 draw complexity와 무관하게 매 frame 전체 division loop 비용을 낸다.
- **가설 A:** conversion SIMD화가 architecture를 유지하는 가장 낮은 위험의 개선이다.
- **가설 B:** dirty region만 변환하려면 unchanged destination의 표현과 다음 frame blending 계약을 먼저 재설계해야 하므로 단순 loop 범위 축소는 안전하지 않다.
- **고위험안:** straight 내부 합성은 alpha 공식 전체를 바꾸므로 별도 RFC가 필요하다.

```text
straight image -> premultiply -> [내부 premultiplied render/composite]
                                   -> full unpremultiply -> straight target
                                       ^ 현재 최적화 후보
```

## 수정 방향과 실현 가능성

1. S/non-S target에서 full/partial scene의 `postRender` 비중과 output pixel 수를 측정한다.
2. scalar `rasterUnpremultiply(uint32_t)`를 bit-exact reference로 alpha 0..255 전체 differential test를 만든다.
3. AVX/NEON/WASM SIMD 또는 reciprocal lookup/vector division prototype의 속도·size·rounding을 비교한다.
4. dirty-only/final-write fusion은 surface representation state diagram을 작성한 뒤 별도 prototype으로 둔다.
5. 내부 straight pipeline 전환은 blend/mask/effect 수식과 API 계약을 다루는 후속 설계 이슈로 분리한다.

## 위험과 검증

- alpha 0/1/254/255와 channel 0/255의 rounding을 scalar와 일치시킨다.
- ABGR/ARGB, image/gradient, mask/blend/effect와 full/partial redraw를 검사한다.
- FPS만 아니라 code size와 conversion time/pixel을 보고한다.
