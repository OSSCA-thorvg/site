# #3840 cpu: graphics are often snapped to the screen-space pixel grid

- Link: https://github.com/thorvg/thorvg/issues/3840
- 난이도: 82/100
- 실현 가능성: 중간
- 초심자 추천: 하위 과제만 조건부 추천
- 관련 영역: SW rectangle fast path, Gaussian kernel, shadow offset, subpixel raster
- 배울 수 있는 것: fixed point, quantization, quality/performance trade-off

## 이슈 요약

rectangle edge, blur radius와 shadow offset이 정수 pixel로 양자화되어 animation이 끊겨 보이는 umbrella 이슈다. 세 현상은 원인과 해법이 달라 하나의 patch로 처리하면 안 된다.

## main 코드 조사

**확인된 증거**

- axis-aligned rect는 `fastTrack=true`가 되며 `utilBBox()`가 bounds를 `round()`한다. 일반 outline은 1/64 fixed-point 좌표를 사용한다.
- `_gaussianInit()`은 float sigma에서 integer box kernel radius를 만든다.
- shadow offset은 float distance 계산 후 `SwPoint`의 `int32_t`로 cast된다.

```cpp
if (fastTrack) {
    renderBox.min = {(int32_t)round(bbox.min.x), (int32_t)round(bbox.min.y)};
}

rd->offset = {(int32_t)(distance * cosf(radian)),
              (int32_t)(distance * sinf(radian))};
```

| 현상 | 양자화 위치 | 독립 해법 후보 |
|---|---|---|
| rect edge | fast-track bbox round | fractional edge coverage/fast-path 조건 축소 |
| blur | integer box kernel | adjacent-kernel blend/fractional filter |
| shadow | integer offset | fractional resampling |

## 원인 가설과 확인 방법

- **확정:** 세 코드 경로가 연속 입력을 integer로 줄인다.
- **가설:** 사용자 체감이 가장 큰 항목은 rect fast path이며 가장 낮은 비용으로 분리 가능하다.
- **확인 방법:** 각 항목을 0.1px step animation으로 나눠 pixel centroid와 frame time을 측정한다.

## 수정 방향 계획

1. rect/blur/shadow를 세 fixture와 하위 issue로 분리한다.
2. fractional rect만 일반 RLE로 보내는 baseline과 edge-aware fast path를 비교한다.
3. shadow는 bilinear shift, blur는 adjacent result blend prototype으로 품질/비용을 측정한다.
4. 이득이 확인된 경로만 개별 patch로 제출한다.

## 실현 가능성 판단

전체는 **중간**이나 초심자는 rectangle fixture/하위 수정만 조건부로 적합하다. blur와 shadow는 성능 설계가 더 어렵다.

## 위험/검증

- fast path 제거에 따른 UI benchmark, SIMD/Grayscale/partial rendering을 확인한다.
- integer coordinate 입력의 기존 pixel은 유지되어야 한다.
- frame time과 memory 증가를 image quality와 함께 보고한다.
