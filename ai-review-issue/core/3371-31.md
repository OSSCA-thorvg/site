## 이슈 요약

Fedora의 Packit 또는 GitHub Actions 안 Fedora container를 이용해 ThorVG commit/PR build를 검사하자는 인프라 제안이다. current repository에는 Ubuntu/Windows/macOS/iOS/Android workflow가 있지만 Fedora job은 없다. Packit은 외부 service 권한과 RPM 정책까지 커질 수 있으므로 첫 단계는 최소 권한의 Fedora container compile/test job으로 제한하는 것이 가장 실현 가능하다.

## main 코드 조사

### 확인된 사실

- [`.github/workflows/build_ubuntu.yml`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/.github/workflows/build_ubuntu.yml)은 shared build, static/threads-off build, unit test와 sanitizer를 별도 jobs로 실행한다.
- Android/iOS/macOS/Windows workflows도 있으나 Fedora 이름/job/container/Packit config는 local tree에 없다.
- Ubuntu full build는 Meson/Ninja와 turbojpeg/png/webp development packages를 설치하고 CPU+GL, loaders/savers/C API/tools를 build한다.
- unit test의 all-engine 구성은 wgpu-native 설치와 software GL package까지 필요하므로 Fedora 첫 job에 그대로 복사하면 범위가 커진다.
- workflow permissions는 기본적으로 `contents: read`이며 Fedora compile job도 추가 write permission이 필요하지 않다.

최소 job의 구조는 다음 정도로 제한할 수 있다.

```yaml
jobs:
  fedora-build:
    runs-on: ubuntu-latest
    container: fedora:<pinned-version>
    steps:
      - uses: actions/checkout@v4
      - run: dnf install -y meson ninja-build gcc-c++
      - run: meson setup build -Dtests=true
      - run: meson test -C build --print-errorlogs
```

### 아직 가설인 부분

- **가설 A:** CPU/default loaders의 minimal job만으로 Fedora compiler/system-header 차이를 조기에 잡는 가치가 있다.
- **가설 B:** system RapidJSON(#3358)이나 RPM build까지 한 job에 넣으면 이슈가 packaging project로 커진다. 후속 matrix로 분리하는 편이 좋다.
- **미확인:** current Fedora package 이름과 Packit onboarding 요구는 이번 local-only 조사에서 조회하지 않았다. 구현 시점에 공식 package/service docs 확인이 필요하다.

## 수정 방향과 실현 가능성

1. Fedora release를 pin하고 CPU/default loader + unit test의 최소 job을 만든다.
2. local Fedora container에서 `dnf` package와 Meson command를 먼저 검증한다.
3. workflow permissions를 `contents: read`로 유지하고 artifact/upload는 필요할 때만 추가한다.
4. CI 시간과 중복을 확인한 뒤 external loaders/system libraries 또는 Packit RPM build를 별도 단계로 논의한다.
5. pinned release 갱신 주기와 failure ownership을 README/comment에 짧게 남긴다.

**판정:** repository 외부 write 없이 local workflow 작성·container test까지 초심자가 수행하기 좋다. 실제 Actions 활성화/Packit 연결은 maintainer가 담당해야 한다.
