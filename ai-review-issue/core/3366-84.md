# #3366 — SVG `feColorMatrix` filter 지원

- Link: https://github.com/thorvg/thorvg/issues/3366
- 난이도: 84/100
- 실현 가능성: 낮음 (parser/model subtask는 중간)
- 초심자 추천: 조건부 (fixture와 parser까지만)
- 분석 기준: `main` working tree `f989b27892ba`
- 관련 영역: SVG filter parser/model, color-matrix effect, CPU/GL/WG compositor
- 배울 수 있는 것: 4×5 RGBA matrix, premultiplied alpha, filter color space

## 이슈 요약

SVG `feColorMatrix`가 무시되어 reference와 색이 다른 compliance 이슈다. current main에는 tag factory, node data, builder conversion과 generic color-matrix RenderEffect가 모두 없다. parser 단계의 누락은 확정적이지만 완전 지원은 `matrix/saturate/hueRotate/luminanceToAlpha`, color space와 three-backend effect implementation까지 필요하다.

## main 코드 조사

### 확인된 사실

- [`SvgNodeType`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/loaders/svg/tvgSvgCommon.h)에는 `GaussianBlur`만 있고 ColorMatrix node나 20 coefficients 저장 구조가 없다.
- [`graphicsTags[]`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/loaders/svg/tvgSvgLoader.cpp)은 `feGaussianBlur`만 filter primitive로 등록한다. `feColorMatrix`는 node를 만들지 못한다.
- [`_applyFilter()`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/loaders/svg/tvgSvgBuilder.cpp)은 GaussianBlur child만 `SceneEffect::GaussianBlur`로 변환한다.
- public/internal Scene effect에는 Fill/Tint/Tritone이 있지만 임의 4×5 RGBA matrix effect는 없다.
- SW/GL/WG는 effect update/render를 각각 type switch로 구현하므로 새 internal effect를 추가하면 세 경로가 필요하다.

모든 SVG type은 결국 4×5 matrix로 정규화할 수 있지만 적용 domain이 중요하다.

```text
| R' |   | m00 m01 m02 m03 m04 | | R |
| G' | = | m10 m11 m12 m13 m14 | | G |
| B' |   | m20 m21 m22 m23 m24 | | B |
| A' |   | m30 m31 m32 m33 m34 | | A |
                                      | 1 |
```

### backend 비교

| 단계 | SW | GL | WG |
|---|---|---|---|
| effect data | `RenderEffect` params | uniform/buffer | uniform buffer |
| execution | compositor pixel loop | fullscreen fragment pass | WGSL compositor pass |
| 핵심 일치 조건 | unpremultiply/matrix/premultiply와 clamp | 같은 순서 | 같은 순서 |

### 아직 가설인 부분

- **확인된 직접 누락:** parser가 tag를 인식하지 않으므로 current main에서 효과가 사라진다.
- **가설 A:** public `SceneEffect`를 늘리지 않고 SVG-internal RenderEffect로 구현할 수 있다. 현재 builder가 public Scene API를 사용하므로 internal 전달 경계 설계가 필요하다.
- **가설 B:** `type="matrix"`부터 단계적으로 지원 가능하지만 unsupported type을 조용히 무시하지 않는 정책이 필요하다.
- **가설 C:** premultiplied pixel에 matrix를 바로 곱하면 alpha가 작은 색에서 reference와 다를 가능성이 높다.

## 수정 방향과 실현 가능성

1. 네 type의 minimal SVG와 numeric expected RGBA fixture를 만든다.
2. parser에서 type/value를 검증하고 모두 canonical 4×5 matrix로 저장한다.
3. sRGB/linearRGB와 straight↔premultiplied 순서를 명세한 CPU reference 구현을 만든다.
4. GL/WG fullscreen effect와 filter region/chaining을 연결한다.
5. transparent color, alpha row, clamp, multiple primitives와 세 backend pixel diff를 검증한다.

**판정:** parser subtask는 분할 가능하지만 issue 전체는 color pipeline과 세 backend가 필요해 실현 가능성이 낮다.
