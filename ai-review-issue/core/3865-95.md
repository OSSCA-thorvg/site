# #3865 gpu: Support Partial Rendering

- Link: https://github.com/thorvg/thorvg/issues/3865
- 난이도: 95/100
- 실현 가능성: 낮음
- 초심자 추천: 비추천
- 관련 영역: RenderDirtyRegion, GL/WG render pass, attachment load/store, composition
- 배울 수 있는 것: dirty rectangle, GPU scissor, retained framebuffer, damage propagation

## 이슈 요약

SW의 SmartRender를 GL/WG에도 적용해 dirty 영역만 clear/draw하려는 최적화다. 공통 region collector는 있지만 GPU의 이전-frame 보존, stencil/MSAA와 intermediate composition을 함께 설계해야 한다.

## main 코드 조사

**확인된 증거**

- `RenderDirtyRegion`은 16 partition, double-buffer list와 old/new bbox 수집을 제공한다.
- SW는 `damage()`로 수집하고 `preRender()`에서 dirty region만 clear한 뒤 교차 영역만 raster한다.
- GL/WG의 `damage()`와 `partial()`은 TODO/no-op이고 public 문서도 SmartRender 미지원을 명시한다.
- GL은 root pass/FBO를, WG는 매 frame root scene/compositor와 command encoder를 자체 관리한다.

```cpp
void GlRenderer::damage(TVG_UNUSED RenderData, TVG_UNUSED const RenderRegion&) { /* TODO */ }
bool GlRenderer::partial(bool) { return false; }
// WgRenderer도 동일한 no-op contract다.
```

| backend | dirty collection | 이전 frame/clear 구조 |
|---|---|---|
| SW | 구현됨 | CPU buffer 부분 clear |
| GL | 없음 | root FBO/render pass, stencil/MSAA |
| WG | 없음 | offscreen root target 후 최종 blit |

## 원인 가설과 확인 방법

- **확정:** common collector는 있으나 GPU integration이 없다.
- **가설:** 핵심 차단점은 scissor API가 아니라 이전 color/stencil/MSAA/intermediate texture 보존 규칙이다.
- **확인 방법:** GL 또는 WG에서 solid shape 이동만 허용한 prototype으로 damage/clear/draw 범위와 full-render pixel을 비교한다.

## 수정 방향 계획

1. 한 backend와 solid, no-mask/effect scene으로 damage collection을 연결한다.
2. root attachment load/store와 dirty clear/scissor, resize/context-loss 시 full redraw 규칙을 정한다.
3. clip/mask/blend/effect에는 우선 safe fallback을 두고 composition bounds 확장 후 점진 지원한다.
4. 두 번째 backend는 첫 contract가 검증된 뒤 구현한다.

## 실현 가능성 판단

제한 prototype은 가능하지만 전체는 **낮음**이다. 초심자는 benchmark/fixture만 맡을 수 있다.

## 위험/검증

- stale stencil/color, MSAA resolve, FBO/texture swap과 final blit을 검사한다.
- full render와 pixel equality, changed-pixel ratio, GPU time/clear bandwidth를 함께 측정한다.
- resize, clear=false, context loss와 WebGL/WASM을 검증한다.
