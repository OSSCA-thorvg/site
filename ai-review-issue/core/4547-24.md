## 이슈 요약

WG engine 변경에서 binary-size diff가 0으로 나오는데 workflow가 WG를 build하지 않는 문제다. unit-test workflow의 WebGPU setup을 참고하라는 구체적인 방향도 있다.

## main 코드 조사

`.github/workflows/binary_size.yml`의 `meson_args`는 `-Dengines="cpu, gl"`만 지정한다. extra package에도 wgpu-native 설치가 없다. 따라서 WG source가 library에 포함되지 않아 WG-only 변경의 size가 0인 것은 코드상 직접 설명된다. `.github/workflows/build_ubuntu.yml`은 별도 `.github/workflows/install_wgpu_native.sh` 후 `-Dengines="all"`을 사용한다.

```yaml
# binary_size.yml의 현재 핵심
-Dengines="cpu, gl"   # wg가 build graph에 없음

# working reference
.github/workflows/install_wgpu_native.sh
-Dengines="all"
```

## 원인 가설

**확인된 원인:** binary-size workflow가 CPU/GL 구성만 측정하도록 고정되어 WG path를 compile하지 않는 것이 원인이다.

## 수정 방향 계획

1. unit-test workflow와 동일한 pinned wgpu-native 설치 단계를 재사용한다.
2. size build의 engine option에 `wg`를 포함한다.
3. cache key에 wgpu version/installer 변화가 반영되는지 본다.
4. WG source에 작은 변화가 있는 test branch에서 non-zero diff와 artifact를 확인한다.

## 위험/검증

수정 범위와 완료 조건이 명확해 좋은 초심자 Issue다. CI 시간·binary 기준의 의미가 바뀌므로 CPU/GL/WG를 한 binary로 측정할지 engine별 artifact로 나눌지는 maintainer와 확인해야 한다. GitHub workflow 수정은 실제 PR 단계에서만 하며 이번 분석은 read-only였다.
