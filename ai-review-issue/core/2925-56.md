# #2925 sw_engine: wrong dashed strokes rendering

- Link: https://github.com/thorvg/thorvg/issues/2925
- 난이도: 56/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 관련 영역: CPU stroker, dash segmentation, SVG stroke, float 경계
- 배울 수 있는 것: path command 순회, dash state machine, cap/close, backend 비교

## 이슈 요약

폭 `0.31`, dash pattern `15 5`인 닫힌 사각형이 SW engine에서 브라우저/WG와 다르게 보이는 렌더링 결함이다. 입력이 작고 SW 전용 불일치라 조사 경로는 비교적 좁지만, dash 상태와 pixel oracle을 다뤄야 한다.

## main 코드 조사

**확인된 증거**

- CPU는 `_genDashOutline()`에서 `RenderPath`를 line/cubic dash 조각으로 다시 만든 후 stroker와 RLE로 보낸다.
- line leftover는 `curLen < 1.0f`, cubic leftover는 `< 0.1f`, main loop는 `DASH_PATTERN_THRESHOLD(0.001f)`를 사용한다.
- `Close`는 독립 seam 처리가 아니라 `_dashLineTo(dash, ptStart, ...)`로 돌아간다.
- GPU 공통 dash 코드도 존재하므로 WG 결과를 상태 전이 비교 대상으로 사용할 수 있다.

```cpp
// src/renderer/cpu_engine/tvgSwShape.cpp
if (dash.curLen < 1.0f && !tvg::zero(len)) {
    dash.curIdx = (dash.curIdx + 1) % dash.cnt;
    dash.curLen = dash.pattern[dash.curIdx];
    dash.curOpGap = !dash.curOpGap;
}
```

```text
MoveTo -> [dash] -> [gap] -> [dash] -> ... -> Close(ptStart)
             curIdx / curLen / curOpGap 상태가 변마다 이어진다
```

## 원인 가설과 확인 방법

- **확정:** line/cubic/loop가 서로 다른 절대 threshold를 사용한다.
- **가설 1:** line의 `1.0f` 전환이 남은 dash를 너무 일찍 소비한다.
- **가설 2:** 닫힌 contour의 마지막 선분과 첫 dash seam 연결이 잘못된다.
- **확인 방법:** 각 command 전후 `curIdx`, `curLen`, `curOpGap`과 생성된 outline을 기록해 GPU 공통 dash segmentation과 첫 차이를 찾는다. stroke raster 단계는 outline이 같을 때만 다음 후보로 조사한다.

## 수정 방향 계획

1. 원본 SVG/C++ 입력을 고정 크기 SW pixel regression으로 만든다.
2. rectangle을 open/closed 두 버전으로 나눠 `_dashClose()` 관련성을 확인한다.
3. threshold를 임의 변경하지 말고 pattern 길이에 대한 상대 epsilon 또는 공통 상수를 prototype해 line/cubic 결과를 맞춘다.
4. odd count, 음/양 offset, 0-length, Butt/Round cap과 여러 scale을 추가 검증한다.

## 실현 가능성 판단

**중간**이다. 후보 함수와 비교 backend가 명확해 초심자가 재현·계측을 할 수 있다. 다만 `1.0f`를 바로 `0.001f`로 바꾸는 식의 수정은 기존 seam 의미를 확인하지 못하므로, 최종 patch에는 렌더링 경험자의 검토가 필요하다.

## 위험/검증

- SW 결과는 exact byte보다 anti-alias tolerance를 둔 image diff가 적절하다.
- WG/브라우저는 oracle 후보지만 좌표/AA 규칙 차이를 감안해 dash topology도 함께 비교한다.
- ASan/UBSan과 기존 SW stroke test를 실행하고 CPU 성능 회귀도 간단히 확인한다.
