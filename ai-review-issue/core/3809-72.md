# #3809 engines: Drop shadows are incorrectly clipped when they originate from offscreen

- Link: https://github.com/thorvg/thorvg/issues/3809
- 난이도: 72/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 관련 영역: effect bounds, culling, offscreen compositor, SW/GL/WG shadow
- 배울 수 있는 것: damage/region 확장, render target, cross-backend tracing

## 이슈 요약

source layer는 화면 밖이지만 offset shadow는 화면 안에 들어오는 경우 shadow가 사라지는 문제다. effect extension 코드는 이미 있으므로 어느 단계에서 source bounds가 먼저 잘리는지 확인해야 한다.

## main 코드 조사

**확인된 증거**

- Lottie DropShadow는 공통 `SceneEffect::DropShadow`로 lowering된다.
- `SceneImpl::bounds()`는 child region에 `effect->extend`를 더한 뒤 viewport와 교차한다.
- SW region은 blur와 signed offset을 방향별로 더하지만 GL/WG는 sigma와 offset 절댓값으로 대칭 확장량을 계산한다.
- SW 구현에는 body가 clipping되면 shadow도 일부 clipping된다는 FIXME가 남아 있다.

```cpp
// src/renderer/tvgScene.h
pRegion.min.x += eRegion.min.x;
pRegion.max.x += eRegion.max.x;
vport = RenderRegion::intersect(vport, pRegion);
```

| backend | region 특성 | 조사 지점 |
|---|---|---|
| SW | signed offset 방향 확장, 관련 FIXME 존재 | source buffer bbox |
| GL | `extend`를 양방향 동일 적용 | FBO viewport/texture 크기 |
| WG | kernel+offset 기반 scalar extend | root/intermediate target AABB |

## 원인 가설과 확인 방법

- **확정:** effect bounds 확장과 최종 viewport intersection은 존재한다.
- **가설:** child/source가 effect 확장 전 viewport에서 제거되거나 composition bbox가 source pixel을 담지 못한다.
- **확인 방법:** child raw bounds → effect extend → Scene vport → target bbox를 같은 fixture에서 기록해 첫 empty/clip을 찾는다.

## 수정 방향 계획

1. 원본을 한 rect+shadow JSON/C++ fixture로 고정하고 세 backend 현 상태를 비교한다.
2. 네 region 값을 trace해 common Scene과 backend target 중 책임 층을 결정한다.
3. source sampling 범위는 보존하고 최종 output에만 viewport clip이 적용되도록 순서를 고친다.
4. 네 offset 방향, blur 0/양수, partial/fully offscreen 회귀를 추가한다.

## 실현 가능성 판단

**중간**이다. 초심자는 fixture와 bounds trace를 맡기 좋지만, target lifetime/크기 수정은 멘토가 필요하다.

## 위험/검증

- 과도한 offscreen allocation과 huge offset을 제한한다.
- SW/GL/WG pixel 및 target bbox를 함께 비교한다.
- shadow 외 GaussianBlur/other SceneEffect의 bounds를 깨뜨리지 않는지 확인한다.
