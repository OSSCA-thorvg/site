## 이슈 요약

Fedora packaging 정책을 위해 bundled RapidJSON 대신 system RapidJSON을 선택할 Meson option을 제공하자는 요청이다. 단순 include path 변경처럼 보이지만 ThorVG의 bundled 1.1.0은 upstream 그대로가 아니다. `tvg::malloc/realloc/free`와 no-op assert가 `rapidjson.h` 안에 직접 patch되어 있으므로 system header 앞에서 같은 configuration macro를 제공해야 memory behavior가 일치한다.

## main 코드 조사

### 확인된 사실

- [`src/loaders/lottie/rapidjson/`](https://github.com/thorvg/thorvg/tree/f989b27892bab31f224f810a54782055eba1e3bc/src/loaders/lottie/rapidjson)은 RapidJSON 1.1.0 전체 header를 vendoring한다.
- [`tvgLottieParserHandler.h`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/loaders/lottie/tvgLottieParserHandler.h)은 `"rapidjson/document.h"`를 직접 include하며 Lottie Meson에는 dependency 선택이 없다.
- bundled `rapidjson/rapidjson.h`는 `tvgCommon.h`를 include하고 `RAPIDJSON_MALLOC/REALLOC/FREE`를 `tvg::` allocator에 연결한다.
- 같은 file은 `RAPIDJSON_ASSERT(x)`를 no-op으로 바꾼다. system 1.1.0 default와 동작이 다르다.
- repository의 external PNG/JPG/WebP loader에는 optional system dependency 탐색 사례가 있지만 RapidJSON용 auto/system/bundled option은 없다.

필요한 config wrapper의 개념은 다음과 같다.

```cpp
#include "tvgCommon.h"

#define RAPIDJSON_MALLOC(size) tvg::malloc(size)
#define RAPIDJSON_REALLOC(ptr, size) tvg::realloc(ptr, size)
#define RAPIDJSON_FREE(ptr) tvg::free(ptr)
#define RAPIDJSON_ASSERT(x)

#include <rapidjson/document.h>
```

```text
-Drapidjson=auto
  ├─ compatible system header 발견 → system include
  └─ 미발견/incompatible          → bundled fallback

-Drapidjson=system  → 없으면 configure error
-Drapidjson=bundled → 현재 동작 보존
```

### 아직 가설인 부분

- **가설 A:** ThorVG-owned config header를 모든 RapidJSON include보다 먼저 사용하면 bundled source patch를 제거하거나 최소화할 수 있다.
- **가설 B:** version 1.1.x만 허용하는 것이 가장 안전하지만 Fedora 제공 version과 current compatibility는 이번 local-only 조사에서 확인하지 않았다.
- **가설 C:** no-op assert를 system mode에서도 유지해야 완전 동일하지만, packaging 측 기대와 maintainer의 diagnostics 정책을 먼저 합의해야 한다.

## 수정 방향과 실현 가능성

1. Meson option을 `auto/system/bundled` 중 어떤 정책으로 노출할지 정한다.
2. allocator/assert macro를 ThorVG config wrapper로 옮기고 parser가 wrapper만 include하도록 한다.
3. Meson에서 compatible system RapidJSON header dependency를 찾고 system mode에서 bundled include path가 앞서지 않게 한다.
4. bundled fallback과 system 1.1.x로 동일 Lottie parse/error/memory tests를 실행한다.
5. static/shared, cross, Lottie-disabled build와 installed consumer에 include가 새지 않는지 확인한다.

**판정:** 빌드 시스템과 include-order를 이해하는 기여자에게 적당한 중간 난도다. allocator macro를 놓친 “컴파일만 성공” patch는 완료가 아니다.
