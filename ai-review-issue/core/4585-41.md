## 이슈 요약

보고는 transformed image가 RLE clip과 함께 texmap path를 탈 때 마지막 span의 `y`를 inclusive 값으로 얻고, 이를 `while (y < yEnd)`의 exclusive upper bound로 사용해 아래쪽 한 scanline을 빠뜨린다고 설명한다. 제안 수정은 마지막 `y`에 1을 더하는 것이다.

이 분석은 [`v0.15.0`의 `_arrange()`](https://github.com/thorvg/thorvg/blob/v0.15.0/src/renderer/sw_engine/tvgSwRasterTexmap.h#L54-L73)에는 정확히 적용된다. 그러나 2026-07-22 current main의 texmap header에는 `_arrange()`, `regionBottom`, `AASpans`와 `image->rle` 분기가 없다.

## main 코드 조사

current [`rasterTexmapPolygon()`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/renderer/cpu_engine/tvgSwRasterTexmap.h#L539-L578)은 image 사각형을 두 triangle로 나누고 `RenderRegion bbox`를 각 polygon rasterizer에 전달한다. segment loop는 bbox의 `max.y`를 exclusive upper bound로 clamp해 `while (y < yEnd)`로 돈다.

RLE image는 current main에서 별도 direct/scaled 함수로 처리된다. [`rasterScaledRleImage()`와 `rasterDirectRleImage()`](https://github.com/thorvg/thorvg/blob/1341c1ebd8c34134f0b4944fe21d192d1c43bb21/src/renderer/cpu_engine/tvgSwRaster.cpp#L1667-L1695)는 RLE span을 직접 순회하며, 보고서의 `_rasterTexmapPolygon(..., region=nullptr)` 호출 구조와 일치하지 않는다.

반면 `v0.15.0`에서는 마지막 span의 `y`를 그대로 `regionBottom`에 넣고, 같은 파일의 segment loop가 strict `< yEnd`를 사용한다. 이 구버전 경로에서는 inclusive/exclusive 불일치가 코드상 확인된다.

## 원인 가설

- **확인:** `v0.15.0`의 RLE branch는 마지막 covered row를 exclusive bound처럼 사용한다.
- **확인:** current main에는 이 branch가 없으므로 제안된 한 줄 patch를 적용할 위치가 없다.
- **미확인:** vanilla current main에서 첨부 설명과 같은 1px gap이 여전히 재현되는지 보고되지 않았다.
- **추론:** downstream vendored copy 또는 구버전 관찰이 current main의 다른 raster path까지 일반화됐을 수 있다. current에도 시각 문제가 있다면 원인은 bbox 생성이나 current RLE clip 교차 계산일 수 있다.

## 수정 방향 계획

1. 이슈의 rotation+clip 최소 예제를 `v0.15.0`과 current main에서 같은 pixel buffer로 실행한다.
2. `v0.15.0`에서만 실패하면 maintenance branch에 `lastY + 1` fix와 regression을 적용하고 current main 이슈는 영향 없음으로 정리한다.
3. current에서도 실패하면 `RenderRegion.max.y`, clip RLE의 마지막 span과 texmap triangle `yi[2]`를 frame dump해 최초로 1이 줄어드는 지점을 찾는다.
4. current code에 구버전 patch를 형태만 바꿔 이식하지 않는다. half-open range 계약을 bbox/RLE helper 이름과 주석으로 명시한다.
5. 수정 후 reference image가 바뀌면 누락 row 복구인지 단순 AA 변화인지 pixel 좌표로 설명한다.

## 초심자 시작 가이드

8×8 RGBA image와 정수 좌표 clip부터 시작하고, output의 각 row alpha 합을 출력한다.

```text
case A: transform 없음 + clip
case B: 15° rotate + clip
case C: skew + clip
case D: clip bottom이 image bottom과 같음/1px 안쪽/1px 바깥
```

각 case에서 expected covered row를 `[minY, maxY)`로 적은 뒤 마지막 row가 포함되는지 비교하면 screenshot만 보는 것보다 off-by-one을 빠르게 찾을 수 있다.

## 위험/검증

- current main과 `v0.15.0` 결과를 분리해 기록한다.
- clip 없는 transformed image, direct/scaled RLE, rotated/skewed texmap을 각각 검사한다.
- bottom뿐 아니라 top/left/right 경계와 음수 좌표·surface 밖 clip을 확인한다.
- bilinear/nearest, AA on/off, alpha composition 조합을 pixel reference와 비교한다.
- 마지막 span이 `INT32_MAX`에 가까운 비정상 입력에서 무조건 `+1`하지 않도록 overflow와 bounds를 검사한다.
- ASan/UBSan으로 buffer 범위를 확인한다.
