# #4052 — 엔진별 얇은 선 품질 차이

- **Link:** https://github.com/thorvg/thorvg/issues/4052
- **난이도:** 64/100
- **초심자 추천:** 비추천(pixel fixture 작성은 조건부 추천)
- **관련 영역:** GL/WG thin stroke fallback, opacity coverage, MSAA
- **배울 수 있는 것:** subpixel stroke, coverage 근사, transform scale와 backend parity
- **조사 기준:** `main@f989b27892bab31f224f810a54782055eba1e3bc`

## 이슈 요약

software에서는 보이는 얇은 grid가 GL/WG surface에서 거의 보이지 않는 품질 차이다.

- **실현 가능성: 중간.** GL/WG 정책 차이는 코드로 확인되지만 reference coverage를 먼저 정의해야 하며, 상수 하나만 높이는 수정은 안전하지 않다.

## main 코드 조사

- GL은 `MIN_GL_STROKE_WIDTH = 1.0f`이며 실제 world stroke가 더 얇으면 1px geometry를 그리고 `strokeWidth / 1px`로 alpha를 낮춘다. 이 로직은 일반 draw와 `GlSolidBatch` 양쪽에 있다.
- WG도 `MIN_WG_STROKE_WIDTH = 1.0f`를 정의하지만 `WgRenderDataShape::updateMeshes()`의 stroke 경로는 원래 stroke width로 tessellate하고 `opacityMultiplier=1.0f`를 유지한다.
- WG의 `MIN_WG_STROKE_ALPHA`는 thin **fill** fallback에서만 사용된다.

| 경로 | SW | GL | WG |
|---|---|---|---|
| 실제 얇은 stroke | scanline coverage | 폭이 1px보다 작으면 1px geometry + `width/1px` alpha | 원래 폭으로 stroker, multiplier 1.0 |
| 너무 얇은 fill | CPU coverage | 1px stroke fallback + 최소 alpha | 1px stroke fallback + 최소 alpha |
| AA | analytic scanline | 4× MSAA | 4× MSAA |

```text
원래 폭 w < 1px
GL: [ 1px geometry ] × alpha(w/1px)
WG: [ w geometry ]   × alpha(1.0)
SW: pixel별 면적 coverage
```

## 원인 가설

**강한 코드 단서:** GL과 WG의 thin-stroke 정책이 일치하지 않으며 WG에는 1px+coverage 보정이 없다. 다만 이슈 본문은 GL도 약하다고 하므로 GL alpha 근사값/MSAA와 target scale도 함께 확인해야 한다.

## 수정 방향과 실현 가능성

1. 0.1~1.5px 선을 정수/반픽셀 좌표에서 그린 세 backend pixel test를 만든다.
2. device transform 후 폭과 총 coverage를 비교한다.
3. WG에 GL과 동등한 최소 geometry+opacity 정책을 prototype하고, GL 상수도 reference에 맞게 조정한다.
4. 총 alpha 합뿐 아니라 교차하는 두 선과 겹치는 parallel 선의 결과도 비교해 over-darkening을 막는다.

## 위험/검증

alpha를 높이면 겹치는 grid 교차점이 진해질 수 있다. solid/gradient stroke, batch/non-batch, DPR/scale, premultiplied target을 검사한다. fixture 작성은 작게 분리할 수 있지만 raster 정책 수정은 초심자 단독 과제로 추천하지 않는다.
