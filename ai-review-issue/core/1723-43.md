# #1723 — infra: add ThorVG 1.x.x to Meson WrapDB

- Link: https://github.com/thorvg/thorvg/issues/1723
- 난이도: 43/100
- 실현 가능성: 높음 (단, 실제 변경 대상은 외부 WrapDB)
- 초심자 추천: 추천
- 분석 기준: `main` working tree `f989b27892ba`; 외부 WrapDB 현황은 이번 조사에서 조회하지 않음
- 관련 영역: Meson packaging, WrapDB, release archive/checksum
- 배울 수 있는 것: Meson subproject, dependency override, reproducible source package

## 이슈 요약

ThorVG 1.x를 `meson wrap install`로 소비할 수 있도록 WrapDB package를 추가하자는 인프라 이슈다. renderer 알고리즘을 바꾸지 않고 packaging과 소비자 연결을 배울 수 있어 초심자에게 상대적으로 적합하다. 다만 `.wrap`/WrapDB metadata의 실제 제출 대상은 ThorVG main이 아니라 외부 `mesonbuild/wrapdb`이며, 이번 재조사는 사용자 지시에 따라 local docs와 ThorVG source만 확인했다. 따라서 현재 WrapDB 등록 여부는 확정하지 않는다.

## main 코드 조사

### 확인된 사실

- 최상위 [`meson.build`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/meson.build)의 project version은 이 snapshot에서 `1.0.0`, major는 `1`이다.
- [`src/meson.build`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/meson.build)는 `thorvg_dep = declare_dependency(...)`를 만들고 `meson.override_dependency('thorvg-' + vmaj, thorvg_dep)`를 등록한다.
- [`inc/meson.build`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/inc/meson.build)는 public header를 `thorvg-1` subdir에 설치하고 pkg-config filebase도 `thorvg-1`이다.
- loader/saver/engine은 [`meson_options.txt`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/meson_options.txt)에서 optional이라 WrapDB default options 정책을 검토할 수 있다.
- local ThorVG tree에는 `subprojects/*.wrap`이 없다. 이는 ThorVG 자신이 WrapDB package를 포함해야 한다는 뜻은 아니다.

subproject 소비를 위한 upstream 준비는 이미 있다.

```meson
thorvg_dep = declare_dependency(
  include_directories: thorvg_inc,
  link_with: thorvg_lib,
)
meson.override_dependency('thorvg-' + vmaj, thorvg_dep)
```

```text
consumer meson.build
   └─ dependency('thorvg-1', fallback: ...)
         └─ WrapDB .wrap downloads immutable release archive
              └─ ThorVG meson.override_dependency('thorvg-1', ...)
```

### 아직 가설인 부분

- **확인된 범위:** upstream ThorVG는 Meson subproject가 dependency object를 돌려줄 구조를 갖췄다.
- **미확인:** WrapDB에 이미 ThorVG package가 생겼는지, package 이름과 current 제출 schema가 무엇인지는 이 local-only 조사로 판단할 수 없다.
- **가설:** 남은 작업은 release URL, source filename, SHA-256, dependency provide name과 default options를 담은 WrapDB metadata 및 consumer test일 가능성이 높다.

## 수정 방향과 실현 가능성

1. 작업을 시작할 때 WrapDB current package 목록과 최신 공식 guide를 읽어 중복/규칙을 확인한다.
2. 불변 release archive URL과 SHA-256을 사용하고 package/provide name을 ThorVG의 `thorvg-1` override와 맞춘다.
3. 최소 consumer에서 `dependency('thorvg-1')`, header include와 단순 `Shape::gen()` link를 검증한다.
4. shared/static 및 loader를 최소화한 default options가 clean fetch에서 동작하는지 확인한다.
5. 변경은 WrapDB 저장소의 review/CI 절차로 제출하고 ThorVG source 변경이 정말 필요한 경우만 별도 범위로 분리한다.

**판정:** 외부 저장소 작업이라는 점만 이해하면 초심자에게 좋은 packaging 과제다. 다만 URL/checksum/schema를 추측해서 작성해서는 안 되며 시작 시점의 공식 규칙 확인이 필수다.
