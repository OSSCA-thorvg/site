## 이슈 요약

한 `<clipPath>` 안의 여러 child를 합집합으로 처리하는 경우와 clipPath에 다시 `clip-path`를 거는 chaining이 잘못 보이는 이슈다. URL 참조 파싱보다 여러 clip child를 하나의 ThorVG `Shape`로 낮추는 표현의 한계가 핵심 후보다.

## main 코드 조사

**확인된 증거**

- `_applyClip()`은 clipPath의 모든 child를 하나의 `Shape* clipper`에 append한다.
- `_appendClipShape()`는 해당 child가 가진 clipPath를 발견하면 같은 aggregate `shape`에 `_applyClip()`을 호출한다.
- `applying` flag는 순환 참조를 막지만 union/intersection 의미 자체를 계산하지 않는다.
- `<use>`는 참조된 basic shape와 두 transform을 합쳐 append하는 별도 경로를 갖는다.

```cpp
// src/loaders/svg/tvgSvgBuilder.cpp
auto clipper = Shape::gen();
ARRAY_FOREACH(p, clipNode->child) {
    if (_appendClipChild(ctx, *p, clipper, vBox, svgPath)) valid = true;
}
paint->clip(clipper);
```

```text
SVG 의미:      (child A ∪ child B) ∩ chainedClip C
현재 lowering: [A subpath + B subpath]라는 Shape 하나에 재귀 clip
               └─ winding에 따라 A∪B와 같다는 보장이 없음
```

## 원인 가설과 확인 방법

- **확정:** 여러 child는 별도 scene/boolean union이 아니라 한 Shape의 subpath로 누적된다.
- **가설 1:** 반대 방향 또는 겹친 subpath가 fill rule에 의해 hole처럼 처리되어 union과 달라진다.
- **가설 2:** child별 chained clip이 aggregate Shape에 적용되어 적용 범위가 넓어진다.
- **확인 방법:** (a) 겹친 두 child만, (b) 한 child와 chained clip만, (c) 둘을 합친 입력으로 나누고 builder가 만든 path/clip tree와 pixel을 비교한다.

## 수정 방향 계획

1. 원본을 multi-child union과 chaining 두 fixture로 분리한다.
2. URL/node 연결과 transform까지 맞는지 확인해 parser를 먼저 배제한다.
3. 단순 방향 정규화로 union을 보장할 수 있는 범위를 실험하되 self-intersection까지 일반화하지 않는다.
4. 일반 해법은 path boolean union과 offscreen mask Scene 중 메모리·opacity·backend 결과를 비교해 선택한다.
5. `objectBoundingBox/userSpaceOnUse`, `<use>`, nested/circular clip 회귀를 추가한다.

## 실현 가능성 판단

재현 분리와 원인 확인은 높지만 전체 수정은 **중간**이다. 일반 union이 필요하면 #2769와 같은 path boolean 문제가 되어 난도가 급상승한다. 초심자는 fixture와 builder tree 조사까지 맡기 좋고, 최종 표현 선택은 멘토가 필요하다.

## 위험/검증

- SW/GL/WG의 clip 결과를 같은 viewport에서 비교한다.
- opacity가 있는 child를 각각 Scene으로 만드는 우회는 SVG clip 의미와 중간 surface 비용을 함께 확인한다.
- circular dependency가 무한 재귀 없이 기존 로그/실패 동작을 유지하는지 검사한다.
