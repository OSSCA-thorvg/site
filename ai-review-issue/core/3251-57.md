# #3251 — `MoveTo + Close` 점 경로의 round cap

- Link: https://github.com/thorvg/thorvg/issues/3251
- 난이도: 57/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 분석 기준: `main` working tree `f989b27892ba`
- 관련 영역: CPU outline, GL/WG stroker, degenerate closed path
- 배울 수 있는 것: zero-length subpath semantics, stroke cap, backend parity

## 이슈 요약

`M 100 100 L 100 100`은 round cap 점으로 처리되지만 `M 100 100 Z`는 아무것도 그리지 않는다는 이슈다. current main의 세 backend 모두 “유효 line/cubic segment가 하나라도 있었는가”를 기준으로 stroke를 생성한다. MoveTo 직후 Close는 closed contour로 표시되지만 direction과 선분이 없어 point-cap helper까지 도달하지 않는다. 입력이 매우 작고 코드 단서가 선명해 이 batch의 좋은 초심자 후보 중 하나다.

## main 코드 조사

### 확인된 사실

- CPU [`_outlineClose()`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/renderer/cpu_engine/tvgSwShape.cpp)은 MoveTo 한 점을 다시 push하여 closed contour를 만든다.
- 그러나 [`strokeParseOutline()`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/renderer/cpu_engine/tvgSwStroke.cpp)은 `last <= first`인 empty contour를 skip하고, 유효 `_lineTo()`가 없으면 `_endSubPath()`를 호출하지 않는다.
- GL/WG `Stroker::run()`은 LineTo/CubicTo에서만 `validStrokeCap=true`로 만들며 Close 뒤에는 false로 둔다.
- GPU `cap()`에는 `firstPt == prevPt`일 때 `roundPoint()`/`squarePoint()`를 호출하는 구현이 이미 있다. `M L(same)`은 이 경로에 도달하지만 `M Z`는 도달하지 않는다.

```text
M P; L P  → validStrokeCap=true → cap() → roundPoint(P)  ✓
M P; Z    → close(); validStrokeCap=false → cap() 없음  ✗
```

### backend별 수정 지점

| backend | 현재 drop 지점 | 재사용 가능한 기능 |
|---|---|---|
| SW | empty/first-point contour skip | `_addCap` 또는 point outline 생성 |
| GL | Close가 `validStrokeCap=false` | `cap()`의 `roundPoint/squarePoint` |
| WG | GL과 같은 state machine | `cap()`의 `roundPoint/squarePoint` |

### 아직 가설인 부분

- **강한 코드 가설:** Close-only contour를 point stroke로 분류하지 않는 공통 의미 공백이 직접 원인이다.
- **미확정 semantics:** Butt cap은 아무것도 그리는 것이 자연스럽지만 Square cap까지 점 사각형을 그릴지는 issue/reference와 합의가 필요하다.
- **가설 C:** loader가 아닌 public `Shape::moveTo(); close()`에서도 같은 결과일 가능성이 높다. 순수 API test로 확인해야 한다.

## 수정 방향과 실현 가능성

1. SVG와 동일한 public Shape test를 CPU/GL/WG에 추가한다.
2. `MoveTo-only`, `M Z`, `M L(same)`, 일반 closed polygon의 Butt/Round/Square 기대표를 합의한다.
3. contour state에 “point 존재/유효 segment 존재/closed”를 구분한다.
4. point-closed + Round/Square일 때 기존 point-cap helper를 호출하고 CPU도 동일 footprint를 만든다.
5. scale/transform, duplicate MoveTo와 dash/trim 회귀를 확인한다.

**판정:** 세 backend test 환경이 있다면 구현 가능성이 높다. 첫 기여자는 GPU 한쪽만 고치지 말고 parity test까지 범위를 잡아야 한다.
