# #3841 SVG: Radial gradients render incorrectly if fx,fy is too far away from cx,cy

- Link: https://github.com/thorvg/thorvg/issues/3841
- 난이도: 74/100
- 실현 가능성: 중간
- 초심자 추천: 조건부 추천
- 관련 영역: radial gradient, SVG Tiny/SVG2 policy, SW/WG/GL
- 배울 수 있는 것: focal circle, quadratic degeneracy, compliance matrix

## 이슈 요약

focal point가 end circle 밖에 있을 때 browser와 다른 결과가 나는 문제다. parser는 값을 읽지만 공통 `correct()`가 focal과 start radius를 의도적으로 clamp한다. 따라서 수학뿐 아니라 목표 SVG 사양 결정이 필요하다.

## main 코드 조사

**확인된 증거**

- parser/builder는 `fx/fy/fr/cx/cy/r`를 보존해 `RadialGradient`에 전달한다.
- `RadialGradientImpl::correct()`가 focal을 `r*(1-PRECISION)` 안으로 이동시키고 `fr`을 축소한다.
- 주석은 SW/WG SVG2 지원 후 이 로직을 제거하고 Lottie 보정을 loader로 옮기라고 명시한다.
- SW/WG는 보정된 값, GL은 더 일반적인 radial case shader를 사용한다.

```cpp
if (this->r - dist < PRECISION) {
    auto scale = this->r * (1.0f - PRECISION) / dist;
    // focal을 end circle 안으로 이동
}
```

| backend | 현재 상태 |
|---|---|
| SW | `correct()` 후 quadratic |
| WG | `correct()` 후 WGSL quadratic |
| GL | source 주석상 SVG2 일반 case 지원 |

## 원인 가설과 확인 방법

- **확정:** outside focal 정보가 raster 전에 clamp되어 browser SVG2 결과와 달라진다.
- **가설:** clamp 제거만 하면 SW/WG에서 NaN/잘못된 root가 발생한다.
- **확인 방법:** 보정 전 값을 진단 build에 넣어 discriminant/root와 GL result가 처음 갈리는 case를 찾는다.

## 수정 방향 계획

1. outside/on-circle, `fr=0`, `fr>=r`, coincident centers를 사양별 fixture로 만든다.
2. 목표 사양과 Lottie compatibility policy를 먼저 결정한다.
3. SW/WG의 degenerate math를 GL과 같은 branch 의미로 맞춘 뒤 clamp를 loader 정책으로 이동한다.
4. Tiny/Lottie 기존 결과와 GL/SW/WG pixel을 회귀 검사한다.

## 실현 가능성 판단

**중간**이다. 초심자는 compliance fixture와 backend comparison을 맡을 수 있지만 수학/정책 변경은 멘토가 필요하다.

## 위험/검증

- NaN/Inf, zero denominator와 GPU precision을 검사한다.
- SVG Tiny와 Lottie radial samples의 호환성을 별도 확인한다.
- #2974와 중복 구현하지 않고 하나의 math plan으로 정리한다.
