# #1329 — sw_engine: shared-edge fill error

- Link: https://github.com/thorvg/thorvg/issues/1329
- 난이도: 71/100
- 실현 가능성: 중간 (재현·계측), 낮음 (안전한 scan-converter 수정)
- 초심자 추천: 조건부
- 분석 기준: `main` working tree `f989b27892ba`
- 관련 영역: CPU scan conversion, winding rule, RLE coverage
- 배울 수 있는 것: NonZero/EvenOdd, shared-edge cancellation, fixed-point rasterization

## 이슈 요약

두 삼각형 subpath가 같은 대각선을 반대 방향으로 공유할 때 내부 seam 또는 잘못된 fill이 나타난다는 CPU 이슈다. 재현 C++가 짧아서 regression test와 instrumentation은 접근하기 좋다. 그러나 shared edge의 signed contribution이 어느 단계에서 어긋나는지 찾아야 하며, scan conversion의 endpoint 규칙을 잘못 고치면 모든 path에 영향을 준다.

## main 코드 조사

### 확인된 사실

- [`tvgSwShape.cpp`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/renderer/cpu_engine/tvgSwShape.cpp)의 `_genOutline()`은 각 subpath contour와 `RenderShape::rule`을 `SwOutline`에 보존한다.
- [`utilExport()`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/renderer/cpu_engine/tvgSwUtil.cpp)은 transform 후 좌표를 `int32_t(value * 64)`로 26.6 fixed-point화한다.
- [`tvgSwRle.cpp`](https://github.com/thorvg/thorvg/blob/f989b27892bab31f224f810a54782055eba1e3bc/src/renderer/cpu_engine/tvgSwRle.cpp)의 `_decomposeOutline()`은 contour마다 시작점으로 닫고 `_lineTo()`를 통해 cell의 signed `area/cover`를 누적한다.
- `_sweep()`은 같은 scanline cell을 순회하며 누적 `cover`와 `area`로 `_horizLine()`을 호출한다.
- `_horizLine()`은 최종 coverage에서 절댓값을 취하고 EvenOdd는 511 folding, NonZero는 255 clamp를 적용한다. 반대 방향 edge의 상쇄는 그 전 cell 누적 단계에서 일어나야 한다.

공유 대각선은 정확히 반대 방향이다.

```text
(10,10) A ───────── B (100,10)
        │          /│
        │   T1   /  │
        │      /    │
        │    /      │
        │  /   T2   │
(10,100) C ───────── D (100,100)

T1 shared edge: B -> C
T2 shared edge: C -> B
기대: 같은 cell에서 signed contribution이 상쇄됨
```

핵심 sweep 식은 current main에 다음 형태로 남아 있다.

```cpp
cover += cell->cover;
auto area = cover * (ONE_PIXEL * 2) - cell->area;
if (area != 0) _horizLine(rw, cell->x, y, area, 1);
```

### 아직 가설인 부분

- **가설 A:** shared edge 두 번의 fixed-point 좌표는 동일하더라도 endpoint 포함/제외 순서 때문에 한 scanline에서 `area/cover`가 완전히 상쇄되지 않을 수 있다.
- **가설 B:** contour별 closing line이나 cell merge ordering이 문제일 수 있다. `_horizLine()`의 absolute/clamp만 보고 fill-rule 버그라고 단정할 근거는 없다.
- **가설 C:** current main에서 이미 증상이 사라졌을 수도 있다. 해당 sample을 golden test로 실행한 증거가 없어 먼저 재현해야 한다.

## 수정 방향과 실현 가능성

1. 이슈의 path를 CPU test로 추가하고 seam pixel 좌표와 expected RGBA를 고정한다.
2. 두 번째 contour의 방향/순서, NonZero/EvenOdd, 하나의 사각형 path를 비교군으로 만든다.
3. shared diagonal이 지나는 scanline에서 `_recordCell` 전후의 `(x, area, cover)`를 기록한다.
4. 최초 차이가 fixed-point export, line decomposition, cell merge, sweep 중 어디인지 확정한다.
5. endpoint/merge 규칙의 최소 수정 후 horizontal/vertical/diagonal edge와 self-intersection 회귀를 돌린다.

**판정:** 재현 test와 cell trace는 초심자에게 좋은 학습 과제다. scan converter 자체의 수정은 raster 알고리즘 경험자의 검토가 필요하다.
