## 이슈 요약

최근 추가된 pattern 구현의 `_buildBaseTile`, `_patternCellRect`, `_patternContentTransform`, `_patternTileGrid` 등에 대한 sample/test coverage를 보강하는 작업이다.

## main 코드 조사

관련 구현은 `src/loaders/svg/tvgSvgBuilder.cpp` 1122행 부근에 연속해 있고, parser 속성은 `tvgSvgLoader.cpp`의 pattern attribute 처리에 모여 있다. 현재 `test/`에서 SVG pattern 전용 test/resource는 검색되지 않아 coverage gap이 실제로 보인다.

```text
<pattern> attrs -> _patternCellRect
                -> _patternContentTransform
                -> _patternTileGrid
                -> _buildBaseTile -> repeated Paint tree
```

## 원인 가설

**확인된 공백:** 기능 구현 뒤 `patternUnits=userSpaceOnUse/objectBoundingBox`, `patternContentUnits`, `patternTransform`, viewBox와 circular reference 조합을 실행하는 fixture가 부족하다.

## 수정 방향 계획

1. 각 helper branch에 대응하는 최소 SVG를 test resource로 만든다.
2. 공개 `Picture::load()`를 통해 parser+builder 전체를 실행한다.
3. 가능하면 pixel hash보다 bounds/성공 여부와 소수의 대표 pixel을 검증한다.
4. circular reference가 hang 없이 skip되는 case도 포함한다.

## 위험/검증

제품 코드를 바꾸지 않고 현재 동작을 이해하며 test부터 기여할 수 있어 좋은 초심자 Issue다. 다만 golden image를 쓰면 backend/AA 차이에 취약하므로 deterministic assertion을 선택해야 한다.
