- Link: https://github.com/thorvg/thorvg/issues/3115

- 초심자 추천: 비추천

## 이슈 요약

Matte3, Pro Levels, Radial Wipe, Displacement Map, Twirl, Mesh Warp, Wavy, Spherize, Puppet 등을 묶은 umbrella 이슈다. 각 효과가 별도의 schema·수학·bounds·backend 구현을 요구하므로 단일 issue라기보다 여러 feature program에 가깝다.

## main 코드 조사

**확인된 증거**

- Lottie model type은 Custom, Tint, Fill, Stroke, Tritone, DropShadow, GaussianBlur만 열거한다.
- parser는 알려지지 않은 `ty`를 모델로 만들지 못하고 unsupported 로그 후 남은 property를 건너뛴다.
- builder는 일부를 `Scene::add(SceneEffect, ...)`, Stroke는 Shape/Mask 조작으로 lowering한다.
- 공통 `RenderEffect`는 GaussianBlur, DropShadow, Fill, Tint, Tritone 중심이다. 고급 distortion용 texture/geometry graph 표현이 없다.
- SW/GL/WG에는 현재 공통 effect 군의 구현만 있으며 GPU-only 추가는 ThorVG backend 일관성을 깨뜨린다.

```cpp
// src/loaders/lottie/tvgLottieModel.h
enum Type : uint8_t {
    Custom = 5, Tint = 20, Fill, Stroke, Tritone,
    DropShadow = 25, GaussianBlur = 29
};
```

| 효과 부류 | 가능한 lowering | 예상 난도 |
|---|---|---|
| Levels/색 보정 | pixel post-process | 높음 |
| Radial Wipe/Matte | mask/composition | 높음 |
| Displacement/Twirl/Spherize | texture resampling | 매우 높음 |
| Mesh Warp/Puppet | animated geometry deformation | 매우 높음 |

## 원인 가설과 확인 방법

- **확정:** parser 누락뿐 아니라 공통 effect type과 backend 구현도 없다.
- **가설:** Matte/단순 색 효과 일부는 기존 SceneEffect/Mask로 lowering할 수 있지만 distortion/puppet은 새 render graph가 필요하다.
- **확인 방법:** 효과별 샘플을 분리해 “기존 primitive로 표현 가능 / 새 pixel effect / geometry effect”로 기술 spike를 수행한다.

## 수정 방향 계획

1. 이 umbrella를 효과 하나당 별도 issue로 분리하고 type id, property order/unit, 최소 animated sample과 reference frames를 둔다.
2. 가장 단순한 색 효과 하나를 parser→model→SceneEffect→SW vertical slice로 구현한다.
3. 공통 effect contract와 bounds/temporary surface를 고정한 뒤 GL/WG를 맞춘다.
4. displacement/warp/puppet은 별도 render-graph/geometry design 문서 이후 진행한다.
5. 각 효과마다 quality, memory, frame time과 unsupported fallback 정책을 기록한다.

## 실현 가능성 판단

개별 단순 효과는 가능하지만 원 이슈 전체는 **낮음**이다. 범위가 닫히지 않았고 서로 다른 기술 문제를 묶고 있다. 초심자는 sample catalog와 parser fixture만 맡을 수 있으며, 그 하위 작업이 전체 issue 완료를 의미하지 않는다.

## 위험/검증

- AE/exporter version별 property index/unit 차이를 fixture로 고정한다.
- 효과 chain 순서, bounds 확대, temporary surface peak memory와 frame latency를 검사한다.
- SW/GL/WG에 동일 의미가 없는 효과는 한 backend만 silently 지원하지 말고 명시적 정책을 둔다.
