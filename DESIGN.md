# OSSCA × ThorVG 디자인 시스템 v2

`src/styles/global.css`가 구현의 기준이며 이 문서는 유지보수 계약을 요약한다. 화면은 라이트
기본의 조용하고 실용적인 대시보드다. 중립 표면으로 정보 밀도를 정리하고, 주황색은 링크,
활성 상태, 포커스와 주요 동작을 알리는 신호로만 사용한다. 넓은 배경색이나 장식용 색 번짐에
주황색을 쓰지 않는다.

## 색상과 테마

모든 컴포넌트는 역할 기반 CSS 변수를 사용한다. 원시 색상은 Lottie와 iframe의 실제 흰색
렌더링 무대처럼 콘텐츠가 요구하는 경우에만 허용한다.

| 역할 | 라이트 기본값 | 다크 값 |
|---|---|---|
| 주요 신호 | `--primary: #9B3600`, `--primary-hover: #742800`, `--primary-press: #5C2000` | `#FF9A5F`, `#FFB083`, `#EE7C3B` |
| 주요 신호 위 글자 | `--on-primary: #FFFFFF` | `#241108` |
| 옅은 주요 신호 | `--primary-subtle: #FFF0E7` | `#3A2418` |
| 기본 글자 | `--ink: #202124` | `#F5F5F6` |
| 보조 글자 | `--ink-muted: #555B64`, `--ink-subtle: #6A717C`, `--ink-faint: #858B94` | `#C0C1C5`, `#9A9DA4`, `#72757D` |
| 캔버스 | `--canvas: #FFFFFF` | `#171719` |
| 중립 표면 | `--surface-1: #F7F7F8`, `--surface-2: #F0F1F2`, `--surface-3: #E7E9EB` | `#1F1F22`, `#28282C`, `#303035` |
| 경계 | `--hairline: #D8DADD`, `--hairline-strong: #B9BEC5`, `--control-border: #737984` | `#39393F`, `#52525A`, `#83838D` |
| 토글 노브 | `--toggle-thumb: #FFFFFF` | `#241108` |
| 포커스 | `--ring: #9B3600` | `#FF9A5F` |

상태색은 두 테마에서 모두 텍스트와 점에 사용하고, 넓은 면을 채우지 않는다.

| 상태 | 라이트 | 다크 | 매핑 |
|---|---|---|---|
| 열림/완료 | `--status-open: #1F6F3D` | `#62D58B` | `open`, `done` |
| 병합 | `--status-merged: #6F42A1` | `#C49BFF` | `merged` |
| 닫힘/오류 | `--status-closed: #B3261E` | `#FF8178` | `closed`, 오류 문구 |
| 대기/초안 | `--status-draft: #626870` | `#B3B5BA` | `todo`, `draft` |
| 진행 중 | `--cat-azure: #1769AA` | `#73B7F2` | `in-progress`, `wip` |

난이도는 `beginner`에 `--cat-teal`(`#0F6D64`/`#69D1C5`), `intermediate`에
`--cat-amber`(`#855600`/`#F2C15D`), `advanced`에 `--cat-violet`
(`#6541A5`/`#B89AF2`)을 쓴다. 색상만으로 의미를 전달하지 않고 항상 한국어 레이블을
함께 표시한다.

초기 `<html data-theme="light">`가 기본이다. 내비게이션 스위치는 `light`와 `dark` 선택을
`localStorage`의 `ossca-thorvg-theme`에 저장한다. `<head>`의 인라인 스크립트는 값이
정확히 `dark`일 때만 다크 토큰을 적용하며 누락·잘못된 값은 라이트로 처리한다.
`theme-color`도 현재 테마와 동기화한다.

## 타이포그래피

- 제목: 자체 호스팅한 `Space Grotesk`, 이후 `SF Pro Display`, `system-ui`, `sans-serif`.
- 본문과 UI: 자체 호스팅한 `Inter`, 이후 `-apple-system`, `system-ui`, `Segoe UI`,
  `Roboto`, `sans-serif`.
- 코드와 식별자: 자체 호스팅한 `JetBrains Mono`, 이후 `ui-monospace`, `SF Mono`,
  `Menlo`, `monospace`.

자체 호스팅 파일은 라틴 글리프를 제공하고 한국어는 운영체제 시스템 글꼴로 폴백한다.
모든 글자 간격은 `0`이다. 음수 자간을 쓰지 않으며 `vw` 같은 뷰포트 비례 글자 크기를
사용하지 않는다. 제목 크기는 고정값과 반응형 구간의 고정값으로만 조절한다.

## 형태와 깊이

- 카드, 버튼, 입력, 탭, 도구 프레임의 최대 모서리 반경은 `8px`다.
- `9999px` pill은 상태 표시, 토글 트랙·노브, 원형 아바타에만 사용한다.
- 태그, 추천, 난이도처럼 텍스트가 긴 요소는 `6px` 또는 `8px` 반경을 사용한다.
- 그라디언트, 장식용 그림자, 떠 있는 페이지 섹션을 사용하지 않는다. 포커스 링처럼 상태를
  전달하는 그림자는 예외다.
- 카드 안에 카드를 넣지 않는다. 카드 배경은 반복 항목이나 실제 도구 경계에만 사용하고,
  섹션 자체는 배경 없는 레이아웃과 구분선으로 조직한다.

## 레이아웃

- 공통 컨테이너는 최대 `1120px`, 좌우 여백은 `24px`다.
- 홈 소개와 대시보드는 배경 없는 레이아웃이다. 추천 이슈와 최근 글 두 열은 상단 구분선으로
  나누며 `760px` 이하에서 한 열로 바뀐다.
- 이슈는 조밀한 단일 목록, 블로그는 최소 `300px` 열의 카드 그리드, 자료 링크는 최소
  `280px` 열의 그리드다. 일정 본문은 `820px`로 제한하고 플레이그라운드는 탭과 도구 패널을
  화면 전체 폭으로 사용한다.
- 필터는 줄바꿈 가능한 실제 입력과 `select`를 사용한다. 긴 제목, 사용자명, 태그는 컨테이너를
  넓히지 않고 줄바꿈한다.
- 탭은 최소 높이 `44px`, 기본 너비 `164px`를 유지하고 좁은 화면에서는 가로 스크롤한다.
- Lottie 플레이어는 최대 `320px × 320px`이며 좁은 화면에서는 가용 너비에 맞춰 줄어들되
  `aspect-ratio: 1 / 1`을 유지한다. 플레이어 패널은 viewport 높이에 맞춰 도구를 중앙에 둔다.
  Viewer와 WebCanvas iframe은 `100vw` 전체 폭과 viewport 기반 높이를 사용한다.

## 컴포넌트와 데이터

- 내비게이션은 높이 `60px`의 sticky 바이며 현재 경로를 주황색 밑줄로 표시한다.
- 홈은 `recommended=true` 이슈를 먼저 배치하고 4개가 안 되면 나머지 이슈로 채운다.
  최신 공개 글은 최대 3개를 조밀한 목록으로 보여 준다.
- 실제 이슈 행은 GitHub를 새 창에서 여는 링크다. `sample=true` 행은 링크 없는 `article`로
  렌더링하고 `샘플 S001`처럼 명시한다. 추천, 난이도, 상태는 각각 텍스트 레이블을 가진다.
- 블로그 카드는 본문에 포함된 첫 번째 이미지·GIF·Lottie를 16:9 영역에 표시한 뒤 GitHub
  작성자, 날짜, 제목, 요약과 태그를 구성한다. 작성자 아바타는 `32px` 고정 크기이며
  카드 hover/focus에서 미디어를 살짝 확대한다. Lottie는 hover/focus에서만 재생하고
  `prefers-reduced-motion`에서는 정지한다.
- 플레이그라운드는 Nav 바로 아래의 얇은 전체 폭 탭 바와 키보드로 이동 가능한 세 탭을 사용한다.
  로컬 Lottie 무대와 두 외부 iframe은 서로 독립된 전체 폭 패널이며 별도의 중첩 카드로
  감싸지 않는다.
- 일정은 공식 링크를 가진 세로 타임라인, ThorVG 자료는 공식·시작하기·도구 그룹의 외부
  링크 목록이다. 화면에 표시할 콘텐츠는 각 데이터 소스에서 가져오며 페이지에 가짜 사용자나
  검증되지 않은 사실을 만들지 않는다.

## 반응형 규칙

- `900px` 이하: 섹션 상하 여백을 `48px`로 줄인다.
- `760px` 이하: 내비게이션을 두 행으로 바꾸고 `플레이그라운드` 메뉴를 `실습`으로
  축약한다. 홈과 이슈 행은 한 열, 페이지 제목은 `2rem`으로 바꾼다.
- `520px` 이하: 홈 마크를 제목 위로 옮기고 홈 이슈 번호와 본문을 한 열로 쌓는다.
- `400px` 이하: 상단 GitHub 버튼을 숨기고 브랜드와 테마를 첫 행, 메뉴를 둘째 행에 배치한다.
- coarse pointer: 버튼, 내비게이션 링크, 탭과 테마 토글의 터치 영역을 최소 `44px`로 한다.

## 접근성

- 첫 포커스에서 본문 건너뛰기 링크를 제공하고 모든 조작 요소에 명확한 `:focus-visible`
  2px 링을 표시한다.
- 필터 결과 수는 `role="status"`, `aria-live="polite"`, `aria-atomic="true"`로 알린다.
- 새 창 링크는 보이는 목적과 한국어 `(새 창)` 레이블을 제공하고 `noopener noreferrer`를
  사용한다.
- 탭은 `tablist`, `tab`, `tabpanel`, `aria-selected`, roving `tabindex`와 방향키·Home·End
  이동을 유지한다.
- `prefers-reduced-motion: reduce`에서는 스크롤과 전환 동작을 최소화하고 Lottie 자동 재생을
  끈다.
- 색상 외에 텍스트, 점, 경계와 선택 상태를 함께 사용하며 동적 콘텐츠가 이전·다음 요소를
  가리거나 밀어내지 않게 한다.
